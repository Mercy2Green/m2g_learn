## 2. Intersection Audit Conclusion

当前 2025–2026 工作可以分别提供：

1. AIGC / world-model / synthetic-video 数据路线；
2. simulation / mobile manipulation benchmark；
3. quadruped/mobile manipulation execution；
4. tool-use / affordance / tool-selection motivation。

直接 L3 证据并非完全不存在：GET-USE 同时包含 simulation、mobile manipulation 和 generalized tool use。但它是稀疏单点：

- 非四足臂；
- 不使用 AIGC 视频；
- 不显式比较 direct manipulation；
- 不输出 `direct / use_tool / reposition / abort-recovery`；
- 未找到公开代码或数据。

因此合理 gap 是：

> 近两年相关线索尚未在四足臂移动操作的 tool-aware mode decision 上闭合：缺少一个可运行 pipeline，用可控仿真/AIGC 数据表示工具必要性、可用性、可达性与合适性，并使 mobile policy 对状态变化做出正确 mode response。

## 3. Ingredient-by-Ingredient Support

### AIGC / simulation ingredient

- RoboTwin 2.0：可运行 simulation generator。
- MobileManiBench：mobile simulation schema，但 release 未确认。
- DreamGen / Video Policy / UWM：generated video/action route。
- RIGVid：generated video 可驱动 tool-like fixed-base actions。

缺口：没有工具必要性 label preservation 和 quadruped mobile pairs。

### Mobile / quadruped ingredient

- SLIM：simulation-to-real quadruped long-horizon manipulation。
- Human2LocoMan：公开 quadruped data/model/interface。
- MoManipVLA / SG-VLA / RAKOMO：reposition、spatial grounding、reachability。
- SAGA：quadruped structured affordance。

缺口：没有 AIGC/sim tool necessity 闭环。

### Tool-aware ingredient

- GET-USE：simulation-based mobile generalized tool use。
- Counterfactual Tool Use：tool-property intervention。
- Robustness-Aware Selection：tool suitability/robustness。
- iTuP：tool-state-conditioned grasp/action。
- Aff-Grasp：可运行 affordance-only。

缺口：fixed-base 为主；不把 base reposition 和 direct action 作为一等 mode。

## 4. Revised Motivation

> 我们不是单独研究 AIGC、mobile manipulation 或 tool use，而是研究如何把三条近两年已有能力连接起来：以现有 simulation/AIGC 项目产生可验证数据，以 mobile/quadruped 工作定义 reachability 与 reposition，以 tool-use 工作定义 availability/suitability，并在现有 Qwen3-VL/GR00T 等 baseline 上评测四类 mode decision。

## 5. Recommended Weekend Question

> 是否把 GET-USE 作为最近的 L3 motivation，以 RoboTwin 2.0/项目 Isaac Sim 作为数据基础，以 Qwen3-VL 作为本周 runnable reasoning baseline，以 Aff-Grasp 作为 affordance-only 对照，并在 Claim 成立后才接 GR00T/openpi 与 quadruped execution？

## 6. Candidate Claims

### Claim A — Three-Way Intersection Evaluation

- 基于 RoboTwin/Isaac simulation 构造 mobile tool-state matched variants。
- 用 Qwen3-VL 测四类 mode。
- Aff-Grasp只提供 affordance input。
- 比较 independent 与 paired counterfactual context/SFT。

### Claim B — Filtered AIGC Extension

- 以 RIGVid/DreamGen/Dream2Fix 为 idea source。
- 只生成 visual variants。
- 必须验证 tool state、reachability cue 和 mode label preservation。

### Claim C — Later Mobile Policy Integration

- GR00T/openpi 作为 action baseline。
- Human2LocoMan state/action interface。
- MoManipVLA/RAKOMO 提供 reposition/reachability conceptual module。

## 7. What Should Not Be Claimed

- GET-USE 已解决四足 tool necessity。
- SAGA 的 affordance 等于 necessity。
- RoboTwin 是 mobile/quadruped baseline。
- Qwen3-VL/GR00T 是 mobile manipulation 文献。
- fixed-base tool papers 自动覆盖 mobile reachability。
- AIGC/world model 已支持 tool-aware。
- 当前存在可直接运行的三元交集 baseline。
