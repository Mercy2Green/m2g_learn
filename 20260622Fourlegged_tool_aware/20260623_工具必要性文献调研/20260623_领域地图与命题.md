# 领域地图与研究命题（Cleaned）

# 1. 项目边界

研究对象不是“机器人能否完成工具任务”，而是移动操作中的四类高层模式判断：

1. `direct manipulate`
2. `use tool`
3. `reposition base`
4. `abort / recovery`

第一阶段目标是判断工具必要性和工具状态对 mode 的影响，不主张完整轨迹生成或完整 VLA tool-use policy。

# 2. Revised Literature Map

| Direction | Representative papers | Existing focus | Relation to our question | Remaining gap |
|---|---|---|---|---|
| Tool selection and use | Counterfactual Tool Use; Robustness-Aware Tool Selection; iTuP | 哪个工具合适、怎么抓、怎样稳定使用 | 证明工具物理属性和状态应影响选择/动作 | 较少把 direct manipulation 作为平等候选并学习 necessity |
| Affordance grounding | RT-Affordance; AffordGrasp; Precise Affordances | 功能区域、关键姿态、task-oriented grasp | 提供 affordance-only 对照 | affordance 不等于必要、可用、可达或值得获取 |
| Failure and counterfactual data | AHA; Dream2Fix | 执行失败检测、解释、恢复 | 提供正负样本和受控扰动方法 | failure label 多为执行结果，不是策略层 tool/no-tool choice |
| General VLA | OpenVLA; Octo | image/language 到动作的通用映射 | 提供 no-explicit-tool-awareness backbone | 缺少 tool-state intervention，不能证明隐式依赖工具状态 |
| Video/world-model policy | UWM; Video Policy; DreamGen; Gen2Act | 利用 action-free/generated video 改善泛化或动作学习 | 提供后续 AIGC/world-model baseline | generated future 不自动等于可靠 action 或 necessity supervision |
| Mobile/quadruped execution | MoManipVLA; UMI on Legs; RoboDuet | waypoint 到 base-arm/WBC 执行 | 支撑系统分层和 embodiment gap | 不定义或证明 tool-awareness novelty |

# 3. Tier 分层及用途

## Tier 1 — Core motivation / novelty-defining

用途：精读并用于构造研究 gap、数据设计和核心反对意见。

- Creative Robot Tool Use by Counterfactual Reasoning
- Robustness-Aware Tool Selection and Manipulation Planning
- Physics-Conditioned Grasping for Stable Tool Use
- RT-Affordance
- AHA
- Dream2Fix

这些工作共同支持：

- 工具属性、接触状态和失败模式会影响决策；
- affordance/how-to supervision 与 necessity supervision 不同；
- 受控反事实和失败样本是可行的数据机制。

它们不共同支持：

- tool necessity 已经被解决；
- AIGC 已被证明能训练 necessity；
- 四足平台本身构成 novelty。

## Tier 2 — Baseline / method comparison

用途：定义组会 taxonomy 和后续完整论文对照，不全部进入第一阶段实验。

- OpenVLA / Octo：无显式 tool-awareness 的通用 policy；
- RoboTool：language/agent/skill-library route；
- AffordGrasp / Precise Affordances：affordance-only route；
- UWM / Video Policy：future video + action route；
- DreamGen / Gen2Act：AIGC/video data route。

## Tier 3 — Background / system support

用途：解释系统接口和移动操作约束，不用于宣称核心 novelty。

- MoManipVLA
- UMI on Legs
- RoboDuet

它们支持 waypoint/WBC 分层、base-arm coordination 和 fixed-to-mobile embodiment gap；它们不支持 tool necessity、AIGC 数据有效性或 tool-state causal claim。

# 4. Corrected Field Gap

## 可安全使用的表述

> 现有机器人工具使用研究主要覆盖工具选择、affordance grounding、task-oriented grasp、tool-use trajectory 和 contact robustness。相较之下，较少工作系统构造同一任务状态下 direct manipulation 与 tool-use 的成对反事实对照，并评测模型能否根据工具的可用性、可达性和合适性改变高层 mode decision。

## 不应使用的表述

- “现有工作没有研究 when to use tools。”
- “tool-awareness 尚无人研究。”
- “四足臂工具使用是全新问题。”
- “AIGC 可以训练 tool necessity。”
- “world model 学会了机器人意图。”
- “生成视频可以直接监督机器人动作。”

# 5. AIGC 的正确位置

| Data source | Primary role | Reliable supervision | Main limitation |
|---|---|---|---|
| Simulation | 生成受控 matched counterfactual pairs | tool/no-tool feasibility、reachability、mode label、action metadata | contact/sim-to-real gap |
| AIGC | 扩展语义、外观、场景组合 | 默认只可信到视觉/语义候选层 | 物理、action、结果和标签一致性不保证 |
| Real robot | grounding、校准和最终验证 | 真实状态与执行结果 | 昂贵且难覆盖失败 |

因此项目应表述为：

> 我们首先研究 structured positive-negative data 是否改善 tool necessity 判断；仿真是可控反事实标签的主要来源，AIGC 是扩展视觉和语义覆盖的候选支撑变量，真实机器人用于 grounding 和最终验证。

# 6. Recommended Minimal Claim

> 在移动操作任务中，成对反事实的 tool/no-tool、usable/unusable、reachable/unreachable 样本，相比同量非配对数据，可以提升 tool necessity 判别，并使模型在工具状态变化时产生更一致的高层 mode decision。

第一阶段 mode：

- direct manipulate；
- use tool；
- reposition base；
- abort / recovery。

第一阶段不输出或不评价完整连续轨迹；最多输出可选的 coarse waypoint 作为辅助，不作为主 claim。

# 7. Three Revised Candidate Claims

## Candidate Claim A: Counterfactual Tool Necessity Learning

* **Main claim:** matched counterfactual pairs 比同量独立正负样本更能提升 necessity classification 和正确 mode switching。
* **Failure mode:** 模型利用任务词、工具出现或目标距离等单一 shortcut，无法区分“工具存在”和“工具必要”。
* **Why existing methods fail:** 工具任务通常预设使用工具；通用成功示范缺少同初态 direct/tool 对照；执行失败数据不直接监督策略选择。
* **Proposed mechanism:** 固定 backbone 的 mode classifier；加入 structured tool state 和 pairwise counterfactual consistency loss。
* **Required data:** 仿真生成 matched pairs；状态包括 direct feasibility、tool availability、tool reachability、tool suitability。
* **Minimal first experiment:** 只做 reach-extension 和 tool reachability 两类干预；训练四个轻量 baseline；不接 WBC、不训练大 VLA。
* **Key metrics:** macro-F1、N0/N1 confusion、mode accuracy、counterfactual consistency、unnecessary-tool rate。
* **Strongest objection:** 模型只学距离阈值或读取 oracle state。
* **How to answer:** 构造距离匹配 hard pairs；加入 wrong/shuffled state；分别报告 image-only、oracle state、noisy estimated state。
* **What not to claim yet:** 不主张完整 tool-use skill、连续轨迹质量、长期执行或真实机器人闭环成功。
* **Venue tendency:** 两周结果用于 go/no-go；扩展到跨任务、真实机器人和 policy 后偏 ICRA / IROS / RA-L。

## Candidate Claim B: AIGC Data Filtering for Tool-Use Decision

* **Main claim:** 对用于 necessity decision 的生成数据，任务结果一致性和 tool-state consistency 筛选比通用视觉质量筛选更有用。
* **Failure mode:** 生成图像/视频视觉合理，但工具状态、任务结果或 label 已改变。
* **Why existing methods fail:** 通用 CLIP/FID/美观指标不检查 direct feasibility、tool availability 或 mode label 是否仍成立。
* **Proposed mechanism:** 先只做静态或短视频 verifier，检查语义、工具状态、任务标签一致性；不直接恢复连续动作。
* **Required data:** 已有仿真 counterfactual states 的 AIGC visual variants；少量人工审核集。
* **Minimal first experiment:** 固定 Candidate A classifier，比较 unfiltered、generic-filtered、state-consistency-filtered AIGC augmentation。
* **Key metrics:** filter precision/recall、label-preservation accuracy、necessity F1、same-count policy gain。
* **Strongest objection:** 提升来自删除难样本或降低数据量。
* **How to answer:** 控制接受样本数量；加入 same-count random subset；报告多样性和各错误类型。
* **What not to claim yet:** 不主张生成视频提供可靠 action supervision，不主张 world model 学会 intent，不主张 AIGC 是核心贡献。
* **Venue tendency:** 若过滤机制与真实下游增益稳定，可偏 CoRL / ICRA；否则作为 Candidate A 的 supporting experiment。

## Candidate Claim C: Tool-State Intervention for VLA / Mode Output

* **Main claim:** 显式 tool state conditioning 能让固定 VLA/backbone 在工具状态干预下产生更正确的 mode change。
* **Failure mode:** image-language-only 模型在工具不可达、损坏或不必要时仍复制训练分布中的 tool-use mode。
* **Why existing methods fail:** 工具状态隐含在像素中，且训练和评测缺少 matched interventions。
* **Proposed mechanism:** 在 frozen/LoRA backbone 上加入 tool-state token 和 mode head；用 Candidate A pairs 训练。
* **Required data:** 仿真图像、结构化 tool state、mode labels；后续加入 estimated state。
* **Minimal first experiment:** 先使用轻量视觉 backbone；只有 Claim A 成立后再替换 OpenVLA/Octo representation。
* **Key metrics:** mode-flip accuracy、counterfactual consistency、calibration、wrong-state sensitivity。
* **Strongest objection:** 结构化状态是 oracle，收益只是额外标签泄漏。
* **How to answer:** wrong/shuffled/noisy state controls；同等维度随机 token；estimated-state setting；保持 backbone 和样本数一致。
* **What not to claim yet:** 不主张 VLA 已学会完整工具使用，不主张轨迹被因果改善，不主张四足真实执行已解决。
* **Venue tendency:** 若加入真实 perception 和闭环执行，可偏 CoRL / RA-L / TRO；仅 mode head 更适合作为前置验证。

# 8. What Should Not Be Claimed Yet

1. 成对反事实数据一定改善完整 action policy。
2. AIGC 数据能替代仿真或真实 action data。
3. world model 能直接学习“何时行动”或“意图”。
4. tool state 已被证明改变连续 trajectory。
5. 四足臂平台本身构成方法贡献。
6. WBC、IsaacSim pipeline、视频生成器或数据集规模是主贡献。
7. 当前证据足以支持 Science Robotics/TRO 级系统 claim。

# 9. 推荐决策顺序

```text
先验证 Claim A
  ├─ 不成立：停止扩展 AIGC/VLA/WBC，重新定义标签和任务
  └─ 成立：
       ├─ 需要数据泛化 -> 做 Claim B
       └─ 需要 policy integration -> 做 Claim C
```
