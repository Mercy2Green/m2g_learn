## 2. 近两年工作已经提供什么

- AIGC/world model 已能生成 robot-relevant video，并在部分项目中恢复动作或联合预测动作。
- RoboTwin 2.0 等平台已提供可配置仿真数据生成和 domain randomization。
- π0.5、GR00T、Gemini Robotics 提供成熟 robot foundation model 路线；Qwen3-VL 提供可运行的视觉推理基线。
- MoManipVLA、SG-VLA、Human2LocoMan 等说明 mobile/quadruped manipulation 需要 base-arm/whole-body 空间表示和专门数据。
- 2025–2026 工具工作已证明 tool geometry、functional region、grasp 和 contact robustness 会改变工具使用质量。

## 3. 仍未被充分提供的能力

近两年工作仍较少同时满足：

1. 同一任务下包含 `direct manipulate` 与 `use tool` 两条可行/不可行路径；
2. 明确标注 tool availability、reachability、suitability；
3. 对工具状态进行 matched intervention；
4. 输出 `direct / use tool / reposition / abort-recovery`；
5. 在四足臂移动操作状态中评测 mode change；
6. 证明 AIGC 数据经过物理、语义和标签一致性过滤后才有决策价值。

安全表述：

> 2025–2026 工具使用、VLA、生成数据和移动操作工作分别覆盖了工具选择、动作生成、数据扩展和 base-arm execution；但尚缺少一个公开、可直接复用的系统同时学习 direct-vs-tool necessity、工具状态干预和四足臂高层 mode decision。

## 4. Revised Motivation

当前最强 motivation 不是“再训练一个 VLA”，而是：

> 现有 2025–2026 foundation policy 和生成数据平台提供了 backbone 与数据基础，但其输入、标签和评测没有显式隔离工具必要性。项目可研究：仿真生成的可控 tool-state counterfactual pairs，加上经过验证的 AIGC 视觉扩展，能否让现有 2025–2026 reasoning/policy baseline 对工具状态变化做出正确 mode decision。

数据职责：

| Source | Role |
|---|---|
| Simulation | 生成 tool/no-tool、usable/unusable、reachable/unreachable 的可控标签和 matched pairs |
| AIGC | 扩展物体外观、环境和语言组合；不能默认提供 action labels |
| Real robot | 验证状态 grounding 和 mode decision；第一阶段不要求完整闭环 |

## 5. Recommended Minimum Question for Weekend Discussion

> 是否以 Qwen3-VL 作为可运行 reasoning baseline、以 π0.5/openpi 或 GR00T 作为后续 policy baseline、以 RoboTwin 2.0/Isaac Sim 作为数据生成基础，构造四类 tool-aware mode 的 matched counterfactual evaluation；AIGC 只在标签一致性过滤后作为视觉扩展？

周末应决定：

1. 第一阶段优先 Qwen3-VL zero/few-shot reasoning，还是直接适配 GR00T/openpi；
2. 仿真底座选择 RoboTwin 2.0 改造还是项目现有 Isaac Sim；
3. 第一批 mode 是否固定为四类；
4. tool suitability 是否第一阶段加入，还是先做 reachability/availability；
5. Claim 成立后优先接 AIGC filtering 还是 quadruped execution。

## 6. Candidate Claims

### Claim A — Existing 2025–2026 Reasoning Baseline + Counterfactual Data

- **Baseline:** Qwen3-VL 2025。
- **Claim:** matched tool-state pairs 比独立场景 prompting/few-shot data 更能提高四类 mode consistency。
- **Modification:** 固定模型；使用结构化 prompt 或轻量 SFT/LoRA；不从零造 backbone。
- **Evidence needed:** same-count unpaired、wrong-state、distance-matched controls。
- **Not yet:** continuous action、完整 VLA 或真实闭环。

### Claim B — Simulation Data Generator for Tool Necessity

- **Baseline:** RoboTwin 2.0 2025 或当前 Isaac Sim project。
- **Claim:** simulation-in-the-loop 可以产生可验证的 direct/tool/reposition/abort labels。
- **Modification:** 增加单变量 tool-state scene variants 和 oracle feasibility checks。
- **Status:** 2025–2026 未找到可直接生成 tool-necessity pairs 的公开 baseline；这是 project-specific dataset extension，不能称为已有 baseline。

### Claim C — AIGC as Filtered Supporting Data

- **Baseline:** DreamGen、Video Policy、UWM 或 Cosmos-Predict2。
- **Claim:** tool-state/label-consistency filtering 后的 generated visuals 比未过滤生成数据更适合辅助 mode reasoning。
- **Modification:** 只扩视觉/语义；不直接采用 pseudo-action。
- **Not yet:** AIGC 生成可靠 robot action 或学会 intent。

## 7. What Should Not Be Claimed

- 现有工作完全没有 tool-awareness。
- AIGC 视频可以直接作为机器人动作标签。
- world model 已学会工具必要性或意图。
- 四足臂平台本身就是 novelty。
- 自定义小分类器是 strong baseline。
- 训练新的通用 VLM 是第一阶段贡献。
- mobile/WBC 工作证明了 tool-aware decision。
