# 导师决策型 PPT 大纲（18 页）

## Slide 1 — Project Definition: What Is Tool-Aware Mobile Manipulation?

**Key message:** 项目研究高层模式判断，不是“机器人拿工具”的系统展示。

- 平台：四足底盘 + 机械臂；
- 状态：任务、目标、工具可用性/可达性/合适性；
- 输出：direct / use tool / reposition / abort-recovery；
- WBC 和局部执行暂作为固定支撑层。

**Suggested figure/table:** 系统边界图，红框只标 mode-decision layer。

## Slide 2 — Tool-Awareness Is Not Tool Recognition

**Key message:** 识别工具不能回答当前是否应该使用工具。

- 识别：这是什么；
- affordance：它能怎么用；
- state：当前是否可用、可达；
- necessity：不用工具是否仍可完成；
- decision：现在应进入哪个 mode。

**Suggested figure/table:** 同一工具在“必要/不必要/不可达/损坏”四个场景中的决策差异。

## Slide 3 — Levels of Tool-Awareness

**Key message:** 研究应明确落在哪一层，避免把不同能力混成 tool-awareness。

- L1 recognition；
- L2 affordance/function；
- L3 state and feasibility；
- L4 necessity；
- L5 action/mode adaptation。

**Suggested figure/table:** 五层阶梯；本项目第一阶段高亮 L3–L4 和离散 L5。

## Slide 4 — Main Research Question Candidates

**Key message:** 三个候选问题有先后依赖，不能同时作为主贡献。

- A：counterfactual necessity learning；
- B：AIGC decision-data filtering；
- C：tool-state-conditioned VLA mode output；
- 推荐先验证 A；
- B/C 由 A 的结果决定。

**Suggested figure/table:** 决策树或 dependency graph。

## Slide 5 — Literature Map Across the Five Levels

**Key message:** 现有工作主要密集在 affordance、selection 和 execution，necessity 的成对评测较少。

- tool selection/use：which/how；
- affordance：where/how；
- failure data：after-failure；
- VLA/world model：general action/future；
- mobile manipulation：execution interface。

**Suggested figure/table:** 论文 × L1–L5 heatmap，避免按年份堆论文。

## Slide 6 — What Existing Works Solve

**Key message:** 已有技术可以支撑本项目，但不能直接替代 necessity learning。

- counterfactual tool properties；
- robust tool selection；
- wrench-aware grasp；
- affordance plans；
- failure synthesis；
- waypoint-to-WBC execution。

**Suggested figure/table:** “已有模块”流水线，每个模块配 1–2 篇 Tier 1/3 论文。

## Slide 7 — What Existing Works Do Not Establish

**Key message:** 缺口是评测和数据结构，不是简单缺一个更大模型。

- direct manipulation 常不在候选集合；
- 缺少 matched tool/no-tool pairs；
- 工具存在常与工具必要高度相关；
- tool-state intervention 很少独立评测；
- generated video validity 不等于 mode-label validity。

**Suggested figure/table:** 已解决/未建立两列表。

## Slide 8 — Why “When to Use Tools” Is Under-Defined

**Key message:** necessity 需要明确反事实：如果不用工具，任务是否可行且成本是否更低？

- direct feasibility；
- tool availability；
- tool reachability；
- tool suitability；
- acquisition/reposition cost。

**Suggested figure/table:** 状态变量到四类 mode 的真值表。

## Slide 9 — Data Routes: Real / Simulation / AIGC / Hybrid

**Key message:** 三种数据源有不同职责。

- simulation：物理参数可控、提供 pair 和 mode label；
- AIGC：扩语义/外观，不默认提供 action label；
- real：grounding、校准、最终验证；
- hybrid：sim label + AIGC appearance + real calibration。

**Suggested figure/table:** 三列 role/strength/risk 表。

## Slide 10 — Positive-Negative Pair Design

**Key message:** 一次只改变一个决策变量，才能抑制 shortcut。

- 同 task instruction 和目标；
- direct reachable ↔ unreachable；
- tool reachable ↔ unreachable；
- tool usable ↔ unusable；
- 按 base scene 分组切分数据。

**Suggested figure/table:** 一个 base scene 的四个 matched variants。

## Slide 11 — Candidate Claim A: Counterfactual Tool Necessity Learning

**Key message:** 先证明 paired supervision 改善离散 mode，不先做完整 VLA。

- B0 image/language；
- B1 + tool state；
- B2 + paired loss；
- B3 shuffled/wrong state；
- 主指标：F1、mode flip、counterfactual consistency。

**Suggested figure/table:** 四个 baseline 的最小差异表。

## Slide 12 — Candidate Claim B: AIGC Filtering

**Key message:** AIGC 是待验证的数据支撑变量，不是默认贡献。

- 生成外观/语义 variants；
- 检查 tool state 和 label 是否保留；
- 比较 unfiltered、generic filter、state filter；
- 控制接受样本数；
- 不恢复连续动作作为第一步。

**Suggested figure/table:** generation → verifier → classifier 的窄 pipeline。

## Slide 13 — Candidate Claim C: Tool-State Intervention

**Key message:** tool-aware 必须通过 intervention 证明，而不是通过 attention 图或成功 demo 宣称。

- 显式 tool-state token；
- tool reachable/usable 状态翻转；
- 检查 mode 是否按预期改变；
- wrong-state control；
- A 成立后再接 OpenVLA/Octo。

**Suggested figure/table:** 同一图像族下 state flip → expected mode flip。

## Slide 14 — Baseline Taxonomy

**Key message:** PPT taxonomy 与两周实验 baseline 必须分开。

- PPT：VLA、prompt、affordance、sim data、AIGC、world model、LLM agent；
- 两周：B0–B3 四个轻量模型；
- 后续：OpenVLA/Octo、UWM、RoboTool；
- Tier 3 系统论文不是 necessity baseline。

**Suggested figure/table:** baseline family × now/later 矩阵。

## Slide 15 — First-Stage Minimal Experiment

**Key message:** 两周只回答一个问题：paired data 是否改善 mode decision？

- 2 类干预；
- 30–50 base scenes；
- 每场景 4 variants；
- frozen encoder + mode head；
- 无 WBC、无连续轨迹、无 AIGC 主实验。

**Suggested figure/table:** 数据规模、模型和输出的单页实验协议。

## Slide 16 — Expected Reviewer Objections

**Key message:** 实验必须主动排除 threshold learning、oracle leakage 和 sample-count effects。

- “只学距离”；
- “tool state 是答案泄漏”；
- “paired loss 只是更多数据”；
- “规则即可解决”；
- “仿真不能说明真实执行”。

**Suggested figure/table:** objection → required control 对照表。

## Slide 17 — Two-Week Action Plan

**Key message:** 每周有明确 go/no-go 输出。

- Day 1–2：冻结 label ontology 和 scene variables；
- Day 3–5：生成 matched dataset 和 group split；
- Day 6–8：训练 B0–B3；
- Day 9–11：hard-pair 和 shuffled-state ablation；
- Day 12–14：结果复核，决定是否进入 AIGC/VLA。

**Suggested figure/table:** 甘特图 + 三个 go/no-go criteria。

## Slide 18 — Decisions Needed from Advisor

**Key message:** 组会目标是锁定问题边界和第一阶段资源，不是批准大系统。

- 是否同意第一阶段只做 mode decision；
- 四类 mode 是否合理；
- 先做 reachability 还是加入 suitability；
- 是否接受仿真为主、真实为后续 gate；
- Claim A 成立后优先做 B 还是 C。

**Suggested figure/table:** 五个待导师选择的 decision boxes。
