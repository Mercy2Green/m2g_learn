# 2025–2026 Tool-Aware Mobile Manipulation Literature and Baseline Survey

## Slide 1 — Project Question

**Key message:** 研究 AIGC + simulation 数据如何支持四足臂 tool-aware mode decision。

- 四类输出：direct / use tool / reposition / abort-recovery；
- 状态：availability、reachability、suitability、direct feasibility；
- 第一阶段不是完整 trajectory 或 WBC；
- 只讨论 2025–2026 工作。

**Suggested figure/table:** 四种相同任务、不同工具状态的场景图。  
**Supporting works:** Qwen3-VL 2025；Human2LocoMan 2025。

## Slide 2 — Tool-Aware Is More Than Tool-Use Demo

**Key message:** 会抓/用工具不能证明会判断是否需要工具。

- tool selection：哪个工具；
- affordance：抓哪里、功能区在哪里；
- tool state：当前是否可用/可达；
- necessity：不用工具是否仍可完成；
- mode：机器人现在应做什么。

**Suggested figure/table:** recognition→affordance→state→necessity→mode 五层图。  
**Supporting works:** Aff-Grasp 2025；iTuP 2025；Robustness-Aware Tool Selection 2025。

## Slide 3 — 2025–2026 Literature Map

**Key message:** 近两年已有 backbone 和数据机制，但分散在六条路线。

- AIGC/world model；
- simulation/synthetic data；
- VLA/foundation model；
- mobile/quadruped manipulation；
- tool-use/affordance；
- failure/counterfactual data。

**Suggested figure/table:** 六方向 × data/model/execution/tool-state 矩阵。  
**Supporting works:** 全部主清单。

## Slide 4 — AIGC / Video / World-Model Methods

**Key message:** 2025 项目已从视觉编辑发展到 video-action joint modeling。

- Video Policy：future video + action decoder；
- UWM：统一 video/action diffusion；
- DreamGen：robot video + pseudo-action；
- EMMA-Visual：多视角视觉迁移；
- Cosmos-Predict2：通用生成基础设施。

**Suggested figure/table:** 输入、生成内容、是否有动作、代码和算力对照。  
**Supporting works:** Video Policy 2025；UWM 2025；DreamGen 2025；EMMA 2025。

## Slide 5 — AIGC Limitations for Tool-Aware Decisions

**Key message:** 生成视觉可用不等于 necessity label 或 action 可用。

- tool/robot geometry drift；
- contact physics 不可信；
- reachability cue 被改变；
- pseudo-action 依赖 IDM；
- 成功样本偏置；
- 必须做 label-preservation filtering。

**Suggested figure/table:** generation error → wrong mode label 映射。  
**Supporting works:** DreamGen 2025；Dream2Fix 2026；Video Policy 2025。

## Slide 6 — Simulation / Synthetic Data Methods

**Key message:** simulation 最适合产生可控 counterfactual labels。

- RoboTwin 2.0：task generator、domain randomization、100k+ trajectories；
- MobileManiBench：Isaac Sim mobile data、300k trajectories；
- GR00T：真实、人类和 synthetic data mixture；
- 可控变量比 photorealism 更关键。

**Suggested figure/table:** RoboTwin/MobileManiBench/项目 Isaac Sim 对照。  
**Supporting works:** RoboTwin 2.0 2025；MobileManiBench 2026；GR00T N1 2025。

## Slide 7 — Missing Simulation Baseline

**Key message:** 未找到公开的 2025–2026 tool-necessity pair generator。

- 现有 generator 没有 direct-vs-tool 标签；
- 没有四类 mode ground truth；
- 没有 quadruped-arm tool-state interventions；
- 项目只能基于现有 framework 做 extension；
- 该 extension 不能冒充 existing baseline。

**Suggested figure/table:** Existing generator fields vs required fields。  
**Supporting works:** RoboTwin 2.0；MobileManiBench。

## Slide 8 — VLA / Robot Foundation Models

**Key message:** 2025–2026 已有成熟 reasoning 与 policy baseline。

- Qwen3-VL：开放、可直接做视觉四类推理；
- π0.5/openpi：开放 action-policy baseline；
- GR00T N1/current N1.7：开放 foundation robot policy；
- Gemini Robotics/ER：强但 restricted；
- 现有模型无原生 tool-necessity output。

**Suggested figure/table:** access、input、output、first-stage/later-stage。  
**Supporting works:** Qwen3-VL 2025；π0.5 2025；GR00T 2025；Gemini Robotics 2025。

## Slide 9 — Mobile Manipulation Methods

**Key message:** mobile setting增加空间状态和 base-arm coordination，但不自动产生 tool-awareness。

- MoManipVLA：waypoint → base-arm optimization；
- SG-VLA：multi-view/depth/history + auxiliary supervision；
- MobileManiBench：mobile simulation benchmark；
- 这些工作主要解决 mobile feasibility。

**Suggested figure/table:** fixed-arm versus mobile action/state diagram。  
**Supporting works:** MoManipVLA 2025；SG-VLA 2026；MobileManiBench 2026。

## Slide 10 — Quadruped-Arm Methods

**Key message:** 近两年四足臂工作提供数据与执行接口，而非 necessity 证据。

- Human2LocoMan：human/robot data、公开代码/模型；
- MLM：multi-task WBC；
- SLIM：simulation-only long-horizon policy；
- 新难点：moving view、whole-body reach、mode transition。

**Suggested figure/table:** quadruped state/action interface。  
**Supporting works:** Human2LocoMan 2025；MLM 2025；SLIM 2025。

## Slide 11 — Tool-Use / Affordance / Selection

**Key message:** 工具状态已被证明影响 how/which，但 direct-vs-tool 仍弱。

- iTuP：task-conditioned grasp physics；
- robustness-aware：工具/配置鲁棒选择；
- Aff-Grasp：功能区监督；
- AffordGrasp：open-vocabulary task-oriented grasp；
- Counterfactual Tool Use：物理属性 intervention。

**Suggested figure/table:** each work × which/how/whether。  
**Supporting works:** 2025–2026 tool papers。

## Slide 12 — Failure / Counterfactual Data

**Key message:** 2025–2026 工作提供可迁移的数据设计，但多数不可直接运行。

- Guardian：procedural planning/execution failures；
- Dream2Fix：generated failure + verification；
- Counterfactual Tool Use：targeted physical interventions；
- 迁移目标：tool-state matched pairs；
- 当前公开代码不足。

**Suggested figure/table:** source failure type → project intervention。  
**Supporting works:** Guardian 2025；Dream2Fix 2026；Counterfactual Tool Use 2026。

## Slide 13 — Directly Reusable Baseline Table

**Key message:** 真正可运行的资源少于概念相关论文数量。

- Qwen3-VL：first-stage reasoning；
- RoboTwin 2.0：simulation generator；
- Aff-Grasp：affordance-only；
- GR00T N1.7：policy smoke test/later baseline；
- Human2LocoMan：quadruped data/execution。

**Suggested figure/table:** code/data/weights/runnable/role 表。  
**Supporting projects:** official repositories。

## Slide 14 — Adaptable and Conceptual Baselines

**Key message:** 必须区分“有代码但重”和“没有代码”。

- Adaptable：π0.5、UWM、Video Policy、DreamGen；
- Restricted：Gemini Robotics；
- Conceptual：iTuP、robustness selection、counterfactual tool use、Dream2Fix、MoManipVLA、SG-VLA；
- conceptual work 不能写成复现实验。

**Suggested figure/table:** A/B/C reusability levels。  
**Supporting works:** baseline matrix。

## Slide 15 — Gap for This Project

**Key message:** 缺的是数据标签与评测连接，不是缺另一个通用模型。

- existing VLM/VLA 无四类 mode；
- simulation 无 tool-necessity pairs；
- AIGC 无可靠 state/action labels；
- mobile/quadruped work无 tool-state intervention；
- tool-use work通常假定使用工具。

**Suggested figure/table:** existing components around a missing center “necessity/mode”。  
**Supporting works:** cross-direction synthesis。

## Slide 16 — First-Stage Tests From Existing Baselines

**Key message:** 本周实验从 Qwen3-VL、Aff-Grasp 和 RoboTwin/Isaac framework 出发。

- Qwen3-VL zero-shot；
- independent versus matched counterfactual few-shot/SFT；
- Aff-Grasp affordance-only input；
- wrong/shuffled tool-state；
- RoboTwin smoke test or existing Isaac Sim extension。

**Suggested figure/table:** baseline/year/code/input/output/modification 表。  
**Supporting works:** Qwen3-VL 2025；Aff-Grasp 2025；RoboTwin 2.0 2025。

## Slide 17 — What Not to Do Now

**Key message:** 避免把两周验证变成大系统工程。

- 不训练新通用 VLM；
- 不从零造 strong baseline；
- 不训练 UWM/Video Policy/DreamGen；
- 不直接使用 AIGC actions；
- 不做完整 WBC/trajectory；
- 不引入旧 agent/skill 路线。

**Suggested figure/table:** now/later/not-in-scope。  
**Supporting works:** compute/access audit。
