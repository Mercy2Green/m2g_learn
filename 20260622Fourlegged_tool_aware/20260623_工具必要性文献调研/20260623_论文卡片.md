## 1. Creative Robot Tool Use by Counterfactual Reasoning

* **Paper:** Creative Robot Tool Use by Counterfactual Reasoning
* **Year / Venue:** 2026 / arXiv preprint
* **Link:** https://arxiv.org/abs/2605.05411
* **Main claim:** 对工具几何和物理特征做反事实干预，可发现工具完成任务的因果属性并迁移技能。
* **Problem:** VLM 语义相似性不能可靠判断非常规物体能否作为工具。
* **Method in one sentence:** VLM 提议候选属性，在 dynamics model 中扰动几何/物理属性做 causal discovery，再用因果 keypoint 迁移技能。
* **Key mechanism:** counterfactual tool generation + simulated interventions。
* **Data:** 仿真动力学实验；reach、scoop、stepping-platform 三类任务。
* **Evidence:** 作者报告因果属性识别比语义/非因果 baseline 提高工具选择和 keypoint transfer 可靠性。
* **Limitation:** 默认需要工具并研究“什么可当工具”；没有 direct/no-tool necessity 决策。
* **Relation to our project:** 最接近反事实工具数据，但我们的变量是“是否需要/是否可用/是否值得用”。
* **Can be baseline?** yes；作为 counterfactual tool-property baseline，不能单独覆盖移动操作。
* **How or when?** 主要是 which/how，不是 strict when。

## 2. Robustness-Aware Tool Selection and Manipulation Planning with Learned Energy-Informed Guidance

* **Paper:** Robustness-Aware Tool Selection and Manipulation Planning with Learned Energy-Informed Guidance
* **Year / Venue:** 2025 / arXiv preprint
* **Link:** https://arxiv.org/abs/2506.03362
* **Main claim:** 工具选择和轨迹应联合最大化接触交互在扰动下的鲁棒性。
* **Problem:** 语义或功能可行的工具不一定在接触扰动下可靠。
* **Method in one sentence:** 学习 energy/caging-based robustness metric，先选工具与关键配置，再优化保持鲁棒性的轨迹。
* **Key mechanism:** minimum escape energy supervision + hierarchical optimization。
* **Data:** 仿真随机 tool-object configurations；三类任务，含真实 scissors hanging。
* **Evidence:** 三任务中选择更鲁棒工具/配置，并在随机力扰动下优于 clearance baseline。
* **Limitation:** 候选集合默认全是工具；不比较直接操作成本和无工具成功。
* **Relation to our project:** 可作为“工具可用/合适”评分的强 baseline。
* **Can be baseline?** yes；在固定工具任务上比较 robustness-only 与 necessity-aware。
* **How or when?** which/how。

## 3. Physics-Conditioned Grasping for Stable Tool Use / iTuP

* **Paper:** Physics-Conditioned Grasping for Stable Tool Use（早期标题 Dynamic Robot Tool Use with Vision Language Models）
* **Year / Venue:** 2025–2026 revision / arXiv preprint
* **Link:** https://arxiv.org/abs/2505.01399
* **Main claim:** 工具抓取应由任务诱导的 wrench、滑移和力矩决定，而非只由语义或几何稳定性决定。
* **Problem:** 选对工具和轨迹仍可能因抓取点无法承受动态交互而失败。
* **Method in one sentence:** VLM grounding + inverse tool-use planning + physics-conditioned grasp scorer。
* **Key mechanism:** trajectory-conditioned wrench estimate and SDG-Net grasp scoring。
* **Data:** simulation-generated grasp/physics supervision + real-robot tool trials。
* **Evidence:** 作者报告降低 induced torque，并在 hammering/reaching 等任务提高成功率。
* **Limitation:** 固定基座，任务已指定工具；不处理工具必要性和 base stance。
* **Relation to our project:** 证明 tool state 必须改变 grasp/action，是 Candidate 3 的关键依据。
* **Can be baseline?** yes；作为 tool-state-to-grasp/action baseline。
* **How or when?** how。

## 4. CoPa: General Robotic Manipulation through Spatial Constraints of Parts with Foundation Models

* **Paper:** CoPa
* **Year / Venue:** 2024 / arXiv preprint
* **Link:** https://arxiv.org/abs/2403.08248
* **Main claim:** foundation model 可通过物体部件的空间约束生成 open-world 6-DoF manipulation poses。
* **Problem:** 高层语言计划难落到精确、可执行的物理位姿。
* **Method in one sentence:** VLM 先定位 task-oriented grasp part，再生成 task-aware part constraints 和后续位姿。
* **Key mechanism:** coarse-to-fine part grounding + spatial constraint solving。
* **Data:** 无额外训练；真实机器人开放物体/指令实验。
* **Evidence:** 展示多类 open-set 和长时序任务的真实执行。
* **Limitation:** 依赖 foundation-model grounding；不判断工具是否必要，不建模工具状态变化。
* **Relation to our project:** 强“语言提示 + part affordance + trajectory”baseline。
* **Can be baseline?** yes；适合作为 modular spatial baseline。
* **How or when?** how。

## 5. Creative Robot Tool Use with Large Language Models (RoboTool)

* **Paper:** Creative Robot Tool Use with Large Language Models
* **Year / Venue:** 2023 / CoRL
* **Link:** https://arxiv.org/abs/2310.13065
* **Main claim:** LLM 可解析隐式物理约束并生成可执行代码完成创造性工具任务。
* **Problem:** 复杂工具任务需要长期规划、参数计算和技能组合。
* **Method in one sentence:** Analyzer–Planner–Calculator–Coder 把语言任务转成技能程序。
* **Key mechanism:** LLM code generation over a predefined skill/API library。
* **Data:** prompt examples、simulation and real-world tasks；不是端到端 policy data。
* **Evidence:** 在多种被设计为无工具不可行的任务中展示执行。
* **Limitation:** agent/skill library 路线；necessity 来自任务设置或语言推理，无学习型 tool/no-tool 对照。
* **Relation to our project:** 必须保留为非端到端 baseline。
* **Can be baseline?** yes；作为 LLM agent + skills。
* **How or when?** 高层可推理“需要工具”，但核心仍是 planner，非 necessity learning。

* **Evidence:** 作者报告在 novel tasks 上显著优于多类中间表示 baseline。
* **Limitation:** affordance plan 描述“怎么做”，不判断工具是否必要、可用或有成本优势。
* **Relation to our project:** 最公平的 affordance-only baseline。
* **Can be baseline?** yes。
* **How or when?** how。

## 9. OpenVLA: An Open-Source Vision-Language-Action Model

* **Paper:** OpenVLA
* **Year / Venue:** 2024 / CoRL
* **Link:** https://arxiv.org/abs/2406.09246
* **Main claim:** 7B 开源 VLA 可从 970k robot demonstrations 学到强 generalist manipulation 并高效微调。
* **Problem:** 先前 VLA 闭源且难以用于下游适配。
* **Method in one sentence:** SigLIP+DINOv2 visual features 与 Llama-2 backbone 融合，自回归预测离散 action tokens。
* **Key mechanism:** large-scale Open X-Embodiment pretraining + LoRA/quantization adaptation。
* **Data:** 970k 真实 robot demonstrations。
* **Evidence:** 作者报告跨 29 tasks 优于 RT-2-X，并优于 from-scratch Diffusion Policy。
* **Limitation:** 成功示范为主；无显式 tool state/necessity objective。
* **Relation to our project:** 可复现的“无工具意识 VLA”首选。
* **Can be baseline?** yes。
* **How or when?** general action learning；无显式 when。

## 10. Octo: An Open-Source Generalist Robot Policy

* **Limitation:** 没有 tool/no-tool label 或 tool-state causal evaluation。
* **Relation to our project:** 比 OpenVLA 更轻量的 policy initialization。
* **Can be baseline?** yes。
* **How or when?** general action learning。

## 13. Unified World Models: Coupling Video and Action Diffusion for Pretraining on Large Robotic Datasets

* **Paper:** Unified World Models (UWM)
* **Year / Venue:** 2025 / arXiv preprint
* **Link:** https://arxiv.org/abs/2504.02792
* **Main claim:** 同一 transformer 内独立控制 video/action diffusion timestep，可统一 policy、forward/inverse dynamics 和 video generation。
* **Problem:** action-free video 难直接用于 imitation learning。
* **Method in one sentence:** joint video and action diffusion with modality-specific noise schedules。
* **Key mechanism:** one model, multiple conditional inference modes。
* **Data:** large multitask robot datasets + action-free video；simulation and real experiments。
* **Evidence:** 作者报告比 imitation learning 更鲁棒，并从 action-free video 获益。
* **Limitation:** 改善 policy 不等于学会 necessity；视觉/contact fidelity 需单独验证。
* **Relation to our project:** world-model/future-prediction baseline 首选。
* **Can be baseline?** yes if compute permits；也可做轻量复现。
* **How or when?** future/action modeling，未直接 when。

## 14. Video Generators are Robot Policies

* **Paper:** Video Generators are Robot Policies
* **Year / Venue:** 2025 / arXiv preprint
* **Link:** https://arxiv.org/abs/2508.00795
* **Main claim:** 共同生成 robot future video 和 action，可从 action-free video 获得更强泛化和样本效率。
* **Problem:** robot demonstration 稀缺，纯 BC 对分布变化脆弱。
* **Method in one sentence:** video diffusion backbone 接 action diffusion head，端到端学习视觉未来和动作。
* **Key mechanism:** generated video as policy representation。
* **Data:** action-free video + robot demonstrations；simulation/real。
* **Evidence:** 作者报告 unseen object/background/task 泛化和少样本收益。
* **Limitation:** 生成相关性不证明工具状态因果性；contact error 可能被 action head 隐藏。
* **Relation to our project:** 检验 future prediction 是否超越普通 VLA 的直接 baseline。
* **Can be baseline?** yes。
* **How or when?** how/future dynamics。

## 15. DreamGen: Unlocking Generalization in Robot Learning through Neural Trajectories

* **Paper:** DreamGen
* **Year / Venue:** 2025 / arXiv preprint
* **Link:** https://arxiv.org/abs/2505.12705
* **Main claim:** 适配 video world model 生成 robot videos，再恢复 pseudo-actions，可大幅扩展行为和环境。
* **Problem:** 真实 teleoperation 难覆盖新任务和新环境。
* **Method in one sentence:** embodiment adaptation → I2V generation → latent action/IDM recovery → policy training。
* **Key mechanism:** neural trajectories and DreamGen Bench。
* **Data:** 少量真实 embodiment data + generated robot videos。
* **Evidence:** 作者报告从单一 pick-place teleoperation 支持人形机器人 22 个新行为，并发现生成 benchmark 与 policy success 相关。
* **Limitation:** pseudo-action 和接触物理可能错误；成功导向生成偏置。
* **Relation to our project:** AIGC 轨迹生成主 baseline，也是筛选研究的主要对照。
* **Can be baseline?** yes，建议缩小规模复现。
* **How or when?** data scaling for how。

## 17. EMMA: Generalizing Real-World Robot Manipulation via Generative Visual Transfer

* **Paper:** EMMA
* **Year / Venue:** 2025 / arXiv preprint

* **Relation to our project:** AIGC 外观增强 baseline，帮助证明“视觉增强 ≠ necessity data”。

## 18. AHA: A Vision-Language-Model for Detecting and Reasoning Over Failures in Robotic Manipulation

* **Paper:** AHA

* **Link:** https://arxiv.org/abs/2410.00371

* **Limitation:** 主要检测执行结果；不产生 tool/no-tool strategy counterfactual。
* **Relation to our project:** 仿真正负样本生成 baseline 和数据 taxonomy 参考。
* **Can be baseline?** yes。
* **How or when?** failure-aware how/recovery。

## 19. Learning Actionable Manipulation Recovery via Counterfactual Failure Synthesis (Dream2Fix)

* **Paper:** Learning Actionable Manipulation Recovery via Counterfactual Failure Synthesis
* **Year / Venue:** 2026 / arXiv preprint
* **Link:** https://arxiv.org/abs/2603.13528
* **Main claim:** 从真实成功示范经生成 world model 合成成对失败—修正 rollouts，可学习可执行恢复轨迹。

## 22. RoboDuet: Learning a Cooperative Policy for Whole-body Legged Loco-Manipulation

* **Paper:** RoboDuet
* **Year / Venue:** 2024 / IEEE Robotics and Automation Letters
* **Link:** https://arxiv.org/abs/2403.17367
* **Main claim:** locomotion 和 manipulation 两个协作 policy 可通过相互作用实现全身 6D tracking 和跨相似四足平台迁移。
* **Problem:** 单独或松耦合控制腿和臂难充分利用全身 workspace。
* **Method in one sentence:** cooperative locomotion/manipulation policies jointly learn whole-body control。
* **Key mechanism:** mutual policy interaction。
* **Data:** large-scale simulation RL + real quadruped deployment。
* **Evidence:** 作者报告 challenging loco-manipulation tasks 上相对 baseline 的成功率提升和 zero-shot transfer。
* **Limitation:** 输入是跟踪目标而非 task/tool semantics；不处理工具状态。
* **Relation to our project:** WBC/local motion baseline，不能作为主要 tool-awareness 对照。
* **Can be baseline?** yes for low-level tracking；no for necessity claim。
* **How or when?** execution only。
