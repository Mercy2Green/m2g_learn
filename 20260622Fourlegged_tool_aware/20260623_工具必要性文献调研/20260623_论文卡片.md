# 2025–2026 论文卡片：AIGC+Sim × Mobile × Tool-Aware Focus

## P0-Intersection Core

### GET-USE

- **Year / link:** 2025 / https://arxiv.org/abs/2510.25754
- **Intersection Level:** `L3: AIGC/sim + mobile/quadruped + tool-aware`
- **Platform / data:** 22-DoF bimanual mobile robot；simulation embodiment extensions + real robot visuomotor data。
- **Tool-aware level:** tool selection + generalized tool-use action。
- **Method:** 先在 simulation 中探索“新 end-effector/工具几何”，再蒸馏到真实视觉策略，从多物体中选择和使用工具。
- **Direct vs tool / necessity:** 考虑最佳工具缺失和多候选选择，但不把 direct manipulation 作为完整对照，也不输出四类 mode。
- **Code/data:** Not found after search。
- **Use for our project:** 当前最接近三元交集的证据；可迁移 simulated embodiment extension 与“工具缺失”设置。
- **Missing dimension:** 不是四足臂；没有 AIGC 视频；无 base reposition/abort mode；不可直接运行。
- **Weekend role:** Main intersection evidence。

> 结论：找到了稀疏 L3 工作，但未找到公开且可复用、同时覆盖四足臂、AIGC/sim counterfactual pairs 和 tool necessity 四类 mode 的 baseline。

### SAGA

- **Year / link:** 2025 / https://arxiv.org/abs/2512.12842
- **Intersection Level:** `L2C: mobile/quadruped + tool-aware, no AIGC/sim`
- **Platform / data:** quadrupedal manipulator；11 个 real-world tasks 和 demonstration data。
- **Tool-aware level:** structured 3D affordance grounding。
- **Method:** multimodal model 将任务目标 grounding 成 3D affordance heatmaps，再条件控制 whole-body policy。
- **Direct vs tool / necessity:** 不显式研究工具必要性。
- **Code/data:** Not found after search。
- **Use:** 证明 quadruped whole-body policy 可以接受结构化 affordance，而不是仅像素输入。
- **Missing dimension:** 没有 AIGC/simulation data route；affordance 不等于 tool state/necessity。
- **Weekend role:** Main intersection evidence（closest L2C）。

## P0-Mobile-Sim Core

### SLIM

- **Year / link:** 2025 / https://arxiv.org/abs/2501.09905
- **Intersection Level:** `L2A: AIGC/sim + mobile/quadruped, no tool-aware`
- **Platform / data:** Unitree Go1 + WidowX-250S；pure simulation RL → real。
- **Generation pipeline:** privileged teacher、visual student、sim-to-real randomization。
- **Tool task / necessity:** 无；search/move/grasp/transport/drop。
- **Use:** 提供四足臂 long-horizon mode hierarchy、reposition 和 sim-to-real setting。
- **Limitation:** 不能证明 tool-aware；工具状态未建模。
- **Weekend role:** Mobile setting evidence。

### MobileManiBench

- **Year / link:** 2026 / https://arxiv.org/abs/2602.05233
- **Intersection Level:** `L2A`
- **Platform / data:** Isaac Sim；2 mobile platforms；300K trajectories；RGB-D-segmentation-state-action。
- **Generation pipeline:** RL expert 自动生成 100+ tasks。
- **Tool task / necessity:** 未报告明确工具任务或 necessity。
- **Code/data:** public release not found after search。
- **Use:** 最接近 mobile simulation data schema；可借鉴多视角、状态和 action annotations。
- **Limitation:** wheeled mobile manipulation；不可直接复用；无工具状态标签。
- **Weekend role:** Mobile setting evidence。

### SG-VLA

- **Year / link:** 2026 / https://arxiv.org/abs/2603.22760
- **Intersection Level:** `L2A`
- **Platform / data:** 13D mobile base-arm-gripper；simulation household rearrangement。
- **Method:** multi-view RGB、depth、history，加 robot pose、joint、affordance、relative pose、segmentation auxiliary supervision。
- **Tool task / necessity:** 无显式工具任务。
- **Code/data:** Not found after search。
- **Use:** 可迁移“结构化 tool state 作为 auxiliary supervision”。
- **Limitation:** 不可运行；affordance head 不是 necessity。
- **Weekend role:** Mobile setting evidence。

### MoManipVLA

- **Year / link:** 2025 / https://arxiv.org/abs/2503.13446
- **Intersection Level:** `L2A`
- **Platform / data:** mobile base + arm；OVMM simulation + limited real data。
- **Method:** fixed-base VLA 输出 EE waypoint，再做 base-arm trajectory optimization。
- **Tool task / necessity:** 无；但直接支持 `reposition base` 接口。
- **Code/data:** Not found after search。
- **Use:** 说明 semantic action 与 mobile feasibility 应分层。
- **Limitation:** 不支持 tool state 或工具获取。
- **Weekend role:** Mobile setting evidence。

### RAKOMO

- **Year / link:** 2025 / https://arxiv.org/abs/2507.19652
- **Intersection Level:** `L2A`
- **Platform / data:** HyQReal + Kinova Gen3；quadruped simulation。
- **Method:** learned reachability margin + KOMO path optimization。
- **Tool task / necessity:** 无工具；只解决 whole-body reachability。
- **Use:** 可迁移到 tool-tip/base reposition feasibility。
- **Limitation:** pure planning adjacency；不能成为第一阶段 tool-aware baseline。
- **Weekend role:** Mobile setting evidence（reachability）。

### Human2LocoMan

- **Year / link:** 2025 / https://arxiv.org/abs/2506.16475
- **Intersection Level:** `L1B: only mobile/quadruped`
- **Platform / data:** LocoMan；human egocentric + robot data；代码、dataset、models见 https://human2bots.github.io/。
- **Method:** embodiment-specific tokenizers/detokenizers + shared transformer。
- **Tool task / necessity:** household manipulation，未提供 tool necessity。
- **Use:** 可复用 quadruped observation/action interface 和 cross-embodiment data design。
- **Limitation:** 不是 simulation/AIGC + tool-aware 证据。
- **Weekend role:** Mobile setting evidence / later execution。

## P0-Tool-Sim Core

### Creative Robot Tool Use by Counterfactual Reasoning

- **Year / link:** 2026 / https://arxiv.org/abs/2605.05411
- **Intersection Level:** `L2B: AIGC/sim + tool-aware, no mobile`
- **Platform / data:** simulated fixed-base tool tasks。
- **Tool-aware level:** tool state/suitability。
- **Method:** VLM 提议物理属性，dynamics model 做 targeted counterfactual interventions。
- **Direct vs tool:** 无；默认研究哪些物体可当工具。
- **Use:** 直接迁移到 reachable/usable/suitable 单变量 intervention。
- **Limitation:** 无代码、无 mobile/base reposition。
- **Weekend role:** Tool-aware motivation。

### Robustness-Aware Tool Selection

- **Year / link:** 2025 paper / ICRA 2026 / https://arxiv.org/abs/2506.03362
- **Intersection Level:** `L2B`
- **Platform / data:** fixed-base simulation + real tool tasks。
- **Tool-aware level:** tool selection、suitability、contact robustness。
- **Method:** learned energy-based robustness score + hierarchical optimization。
- **Direct vs tool:** 无；候选集合是工具。
- **Use:** 说明“合适”必须包含物理鲁棒性。
- **Limitation:** 无公开实现；无 mobile reachability。
- **Weekend role:** Tool-aware motivation。

### Physics-Conditioned Grasping / iTuP

- **Year / link:** 2025 / https://arxiv.org/abs/2505.01399
- **Intersection Level:** `L2B`
- **Platform / data:** fixed-base simulation supervision + real tasks。
- **Tool-aware level:** tool-use action and task-conditioned grasp。
- **Method:** VLM grounding、trajectory planning、wrench/torque/slip-conditioned grasp scoring。
- **Direct vs tool:** 无；工具使用已给定。
- **Use:** 支撑 tool state 应改变 downstream action，而非仅识别。
- **Limitation:** 无代码；固定基座。
- **Weekend role:** Tool-aware motivation。

### RobotSmith

- **Year / link:** 2025 / https://arxiv.org/abs/2506.14763
- **Intersection Level:** `L2B`
- **Platform / data:** fixed-base physics simulation + real validation。
- **Tool-aware level:** generative tool design and tool-use trajectory。
- **Method:** VLM proposal + simulation optimization。
- **Direct vs tool:** 任务被定义为需要设计工具。
- **Use:** 说明 simulation 可以验证工具几何和使用策略。
- **Limitation:** agent-heavy、无 mobile、无 necessity；无公开实现。
- **Weekend role:** Tool-aware motivation。

### RIGVid

- **Year / link:** 2025 / https://arxiv.org/abs/2507.00990
- **Intersection Level:** `L2B`
- **Platform / data:** generated videos + fixed-base real robot。
- **Tool-aware level:** tool-use action（wiping/mixing/pouring），非 necessity。
- **Method:** video diffusion → VLM filter → 6D object trajectory → robot retargeting。
- **Use:** 最接近 “AIGC video + tool-like manipulation action” 的邻近证据。
- **Limitation:** 无 mobile；不判断工具状态；代码未找到。
- **Weekend role:** AIGC/sim data evidence。

## P0-Runnable Baseline

### Qwen3-VL

- **Intersection Level:** `Remove`（不作为交集证据）。
- **运行状态:** code + open weights；可本周运行。
- **Input / output:** image/video + prompt → structured text。
- **本周用途:** 仿真截图/状态描述下输出四类 mode。
- **不能证明:** mobile execution、tool skill、action feasibility。
- **Weekend role:** Runnable baseline。

### RoboTwin 2.0

- **Intersection Level:** `L1A: only AIGC/sim`。
- **运行状态:** MIT code、100K+ trajectories、collection scripts。
- **Input / output:** task/scene config → multimodal trajectories。
- **本周用途:** generator smoke test；评估加入 matched tool-state variants 的成本。
- **不能证明:** mobile/quadruped tool-aware。
- **Weekend role:** Runnable baseline。

### Aff-Grasp

- **Intersection Level:** `L1C: only tool-aware`。
- **运行状态:** code、HF data、HF model。
- **Input / output:** RGB/video → functional/graspable masks。
- **本周用途:** affordance-only 对照。
- **不能证明:** tool reachability、necessity、reposition。
- **Weekend role:** Runnable baseline。

### GR00T N1.7

- **Intersection Level:** `L1A: only AIGC/sim`（foundation pretraining 包含 synthetic data）。
- **运行状态:** official demo、weights、finetuning scripts。
- **Input / output:** image+state+language → actions。
- **本周用途:** schema/inference smoke test；不是核心实验。
- **不能证明:** 四类 mode 或 mobile tool-awareness。
- **Weekend role:** Runnable baseline / later policy。

## Recommended Reading Order

### Read Group 1: Intersection / closest-to-project

1. GET-USE — 唯一明确 L3；用于 PPT gap 页；不要 overclaim quadruped/necessity。
2. SAGA — L2C quadruped+affordance；用于结构化状态页；缺 AIGC/sim。
3. SLIM — L2A sim+quadruped；用于 mobile data/execution 页；缺 tool-aware。
4. RIGVid — L2B AIGC+tool-use action；用于 AIGC 邻近证据；缺 mobile/necessity。

### Read Group 2: AIGC/sim data for robot learning

- RoboTwin 2.0、MobileManiBench、DreamGen、Video Policy、UWM、Cosmos。
- 用于 PPT 数据路线与 baseline 可执行性。
- 不要声称它们已支持 tool-aware。

### Read Group 3: Mobile/quadruped manipulation

- Human2LocoMan、SLIM、MoManipVLA、SG-VLA、RAKOMO。
- 用于 state/action/reposition/base-arm constraints。
- 不要声称它们证明 tool necessity。

### Read Group 4: Tool-aware/tool-use/affordance

- Counterfactual Tool Use、Robustness-Aware Tool Selection、iTuP、Aff-Grasp、RobotSmith。
- 用于 tool state/suitability/action motivation。
- 不要声称 fixed-base 结论自动覆盖 mobile。

### Read Group 5: Runnable baseline

- Qwen3-VL、RoboTwin 2.0、Aff-Grasp、GR00T N1.7。
- 用于周末的“这周能跑什么”。
- 不要把 baseline 可运行性写成交集证据。
