# 2025–2026 论文卡片

## P0 — 周末必须讲 / 近期可尝试

### 1. Qwen3-VL

* **Paper / project:** Qwen3-VL
* **Year:** 2025
* **Venue / status:** Technical report; open-weight model
* **Link:** https://arxiv.org/abs/2511.21631
* **Code:** https://github.com/QwenLM/Qwen3-VL
* **Dataset:** General multimodal pretraining data未公开；本项目使用自建仿真 evaluation set
* **Model weights:** https://huggingface.co/Qwen/Qwen3-VL-8B-Instruct
* **Main problem:** Image/video spatial and multimodal reasoning
* **Method:** Open multimodal instruction/thinking models with grounding and video understanding
* **Input:** Image/video + text prompt
* **Output:** Text/structured reasoning
* **Training / data requirement:** Transformers inference可直接使用；LoRA/SFT代码公开
* **Relation:** 可作为四类 mode 的第一阶段成熟 VLM reasoning baseline
* **Can be directly used as baseline?** Yes
* **If baseline, how exactly:** 固定模型，以相同 scene image 和结构化 prompt 输出四类 mode；比较 zero-shot、few-shot、matched-pair SFT
* **Limitation:** 不是 robot action policy，不能证明可执行性
* **Priority:** P0

### 2. RoboTwin 2.0

* **Year:** 2025
* **Venue / status:** arXiv preprint; active open platform
* **Link:** https://arxiv.org/abs/2506.18088
* **Code:** https://github.com/robotwin-Platform/robotwin
* **Dataset:** 100k+ trajectories on Hugging Face, linked by repository
* **Model weights:** 多种 policy baseline 支持；不是单一模型
* **Main problem:** Scalable simulation data generation and robust manipulation benchmarking
* **Method:** Task-code generation、simulation verification、expert rollouts、domain randomization
* **Input:** Task configuration, robot embodiment, scene assets
* **Output:** RGB/depth/state/action trajectories
* **Training / data requirement:** 官方文档称安装约 20 分钟；支持 `collect_data.sh`
* **Relation:** 最接近可直接改造的 2025 simulation data baseline
* **Can be directly used as baseline?** Partial
* **If baseline, how exactly:** 复用 task/config/data collection pattern，增加 direct/tool/reachable/usable variants；若 embodiment 不匹配则只迁移生成协议
* **Limitation:** 双臂桌面平台，不是四足臂；当前无 necessity task
* **Priority:** P0

### 3. GR00T N1 / Isaac-GR00T N1.7

* **Year:** 2025–2026 current release
* **Venue / status:** GR00T N1 arXiv 2025; N1.7 current open project
* **Link:** https://arxiv.org/abs/2503.14734
* **Code:** https://github.com/NVIDIA/Isaac-GR00T
* **Dataset:** Demo/LeRobot v2 adapters; DROID/LIBERO integrations
* **Model weights:** https://huggingface.co/nvidia/GR00T-N1.7-3B
* **Main problem:** Generalist cross-embodiment robot action generation
* **Method:** Vision-language reasoning conditioned continuous action model
* **Input:** Images/video + state + language + embodiment config
* **Output:** Continuous action chunks
* **Training / data requirement:** LeRobot v2 + modality config；fine-tuning约 40GB VRAM recommended
* **Relation:** 可运行的 2025 robot foundation model；适合作后续 mode/action baseline
* **Can be directly used as baseline?** Partial
* **If baseline, how exactly:** 首先做 open-loop prompt/state inference；后续添加四类 mode auxiliary target或分层调用
* **Limitation:** 人形/机械臂数据为主；第一周适配成本高
* **Priority:** P0 for survey, P1 for experiment

### 4. Learning Precise Affordances from Egocentric Videos for Robotic Manipulation

* **Year:** 2025
* **Venue / status:** ICCV 2025
* **Link:** https://openaccess.thecvf.com/content/ICCV2025/html/Li_Learning_Precise_Affordances_from_Egocentric_Videos_for_Robotic_Manipulation_ICCV_2025_paper.html
* **Code:** https://github.com/Reagan1311/Aff-Grasp
* **Dataset:** https://huggingface.co/datasets/Reagan1311/Data_for_Aff-Grasp
* **Model weights:** https://huggingface.co/Reagan1311/Model_for_Aff-Grasp
* **Main problem:** Precise graspable and functional affordance learning
* **Method:** Mine hand-object regions from egocentric video and train an affordance predictor
* **Input:** RGB/interaction video
* **Output:** Functional and graspable masks
* **Training / data requirement:** Public extraction and learning modules
* **Relation:** 唯一较完整可复用的 2025 affordance-only baseline
* **Can be directly used as baseline?** Partial
* **If baseline, how exactly:** 将 functional mask/score 作为工具功能信息，要求 Qwen3-VL或后续 policy只基于 affordance做 mode判断
* **Limitation:** 不输出 availability、reachability 或 necessity
* **Priority:** P0

### 5. Physics-Conditioned Grasping for Stable Tool Use

* **Year:** 2025
* **Venue / status:** arXiv preprint / under review
* **Link:** https://arxiv.org/abs/2505.01399
* **Code / Dataset / Weights:** Not found after search
* **Main problem:** Dynamic tool-use grasp stability
* **Method:** VLM grounding + inverse tool-use planning + physics-conditioned grasp scoring
* **Input:** Scene, task, tool, trajectory and grasp candidates
* **Output:** Tool-use trajectory and grasp
* **Training / data requirement:** Simulation physics supervision
* **Relation:** 证明 tool state/interaction physics 应改变 downstream action
* **Can be directly used as baseline?** No
* **If baseline, how exactly:** 只能作 conceptual tool-usability score；不能宣称复现
* **Limitation:** 工具使用已给定，固定基座
* **Priority:** P0 motivation

### 6. Robustness-Aware Tool Selection and Manipulation Planning

* **Year:** 2025 paper / ICRA 2026
* **Venue / status:** ICRA 2026
* **Link:** https://arxiv.org/abs/2506.03362
* **Code / Dataset / Weights:** Not found after search
* **Main problem:** Tool/configuration selection under contact disturbances
* **Method:** Learned energy-based robustness metric + hierarchical optimization
* **Input:** Candidate tools/configurations and environment
* **Output:** Tool selection and manipulation trajectory
* **Relation:** 支撑 tool suitability 与 robustness 不应被简化为识别
* **Can be directly used as baseline?** No
* **If baseline, how exactly:** 后续可实现“direct action也作为候选”的简化 score baseline；本周只作 motivation
* **Limitation:** 没有 direct/no-tool decision；无公开实现
* **Priority:** P0 motivation

### 7. Creative Robot Tool Use by Counterfactual Reasoning

* **Year:** 2026
* **Venue / status:** arXiv preprint / under review
* **Link:** https://arxiv.org/abs/2605.05411
* **Code / Dataset / Weights:** Not found after search
* **Main problem:** Discover causal physical properties for unconventional tool use
* **Method:** VLM-proposed attributes + dynamics-model counterfactual interventions
* **Input:** Candidate object/tool/task properties
* **Output:** Causal suitability and transfer keypoints
* **Relation:** 最接近 tool-state intervention 的 novelty source
* **Can be directly used as baseline?** No
* **If baseline, how exactly:** 迁移 intervention protocol 到 reachable/usable/suitable variants
* **Limitation:** 研究哪个物体能当工具，不研究是否需要工具
* **Priority:** P0 motivation

### 8. Human2LocoMan

* **Year:** 2025
* **Venue / status:** RSS 2025
* **Link:** https://arxiv.org/abs/2506.16475
* **Code / Dataset / Weights:** linked at https://human2bots.github.io/
* **Main problem:** Data-efficient quadrupedal manipulation with human pretraining
* **Method:** Modular cross-embodiment transformer with embodiment-specific tokenizers/detokenizers
* **Input:** Egocentric vision, proprioception, language/task context
* **Output:** Unimanual/bimanual robot actions
* **Training / data requirement:** Human + LocoMan datasets and released models
* **Relation:** 当前最完整的 2025 quadruped manipulation data/model reference
* **Can be directly used as baseline?** Partial
* **If baseline, how exactly:** 复用数据接口/模型作为后续 execution baseline；第一阶段只用于定义 quadruped state
* **Limitation:** 无 tool necessity labels
* **Priority:** P0 setting

## P1 — 后续完整论文 baseline

### 9. π0.5 / openpi

* **Year:** 2025
* **Venue / status:** CoRL 2025; official openpi project
* **Link:** https://arxiv.org/abs/2504.16054
* **Code:** https://github.com/Physical-Intelligence/openpi
* **Dataset:** LeRobot/DROID/LIBERO/custom conversion
* **Model weights:** π0.5 base and task checkpoints in repository configs
* **Main problem:** Open-world generalist manipulation
* **Input:** Multi-view images + state + language
* **Output:** Action chunks
* **Training / data requirement:** LoRA >22.5GB; full tuning >70GB reported by repo
* **Relation:** 强 VLA policy baseline
* **Can be directly used as baseline?** Partial
* **If baseline, how exactly:** Claim A通过后，加入 mode token/auxiliary prediction或按 mode 分流 action policy
* **Limitation:** embodiment/action mapping工作量大
* **Priority:** P1
