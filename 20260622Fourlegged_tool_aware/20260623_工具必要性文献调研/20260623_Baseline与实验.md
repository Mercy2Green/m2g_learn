# Part 3: Baseline 建议

## Baseline 1：不使用工具意识的 VLA

* **做什么：** 微调 OpenVLA 或 Octo，输入 RGB(+proprioception)+language，直接输出 EE/action chunk；训练数据可含工具任务，但没有 necessity label、tool state token 或 mode head。
* **为什么公平：** backbone、训练步数、机器人数据量与主方法一致，只移除 tool-awareness supervision。
* **检验 claim：** 通用 VLA 是否已能从像素隐式学会 when-to-use。
* **需要数据：** 成功 demonstrations，混合 direct 和 tool-use tasks。
* **最小实验：** N0–N3 测试；报告 direct/tool mode 混淆、task success 和 unnecessary tool use。

## Baseline 2：只用语言提示 tool-use

* **做什么：** 同一 VLA，prompt 显式写“use the hook”或“do not use tools”；或让 VLM 先输出 use/no-use 文本，再执行。
* **为什么公平：** 检验收益是否只是把答案写进语言，而非感知工具状态。
* **检验 claim：** 状态条件和反事实监督是否优于 prompt engineering。
* **需要数据：** 同一轨迹的语言改写；无需额外状态标注。
* **最小实验：** 比较 explicit prompt、neutral instruction、错误 prompt；测试 prompt 与真实工具状态冲突时的行为。

## Baseline 3：只用 affordance recognition

* **做什么：** RT-Affordance/AffordGrasp 风格预测工具 mask、functional part、grasp point 或 affordance waypoint，再交给相同 controller。
* **为什么公平：** 它拥有“工具在哪里/怎么抓”的信息，但没有“是否值得/必须使用”标签。
* **检验 claim：** tool necessity 是否超出 affordance localization。
* **需要数据：** mask/keypoint/pose annotations；可由仿真自动生成。
* **最小实验：** 保持 perception 和 controller 一致，只比较 affordance-only 与 affordance+necessity。

## Baseline 4：使用 AIGC 正负样本训练

* **做什么：** 从成功轨迹生成视觉或视频变体，并生成/编辑失败样本；不使用 proposed robot-specific verifier，或只用 CLIP/video-quality 过滤。
* **为什么公平：** 相同生成模型、相同样本数、相同 policy；只改变筛选和 pair construction。
* **检验 claim：** AIGC 本身是否足够；proposed filtering/counterfactual pairing 是否必要。
* **需要数据：** 真实/仿真 seed trajectories、生成 prompts、伪动作或保留动作。
* **最小实验：** real only、real+AIGC random、real+AIGC generic-filter、real+AIGC robot-filter。

## Baseline 5：使用仿真正负样本训练

* **做什么：** 在 IsaacSim 中随机化工具存在、位置、尺寸、破损、目标距离和 base pose；用独立样本训练 binary/multiclass classifier。
* **为什么公平：** 仿真提供准确物理标签，是低噪声上限和 AIGC 的对照。
* **检验 claim：** paired counterfactual 是否优于 unpaired random negatives；AIGC 是否提供超出 simulation 的视觉多样性。
* **需要数据：** scene parameterization、success oracle 或 planner/rollout labels。
* **最小实验：** unpaired random negatives vs matched counterfactual pairs，控制总样本数和类平衡。

## Baseline 6：world model / future prediction

* **做什么：** UWM/Video Policy 风格预测 direct 和 tool-use 两种候选未来，再按 goal score 选动作；或联合 future-video/action training，但不加 necessity loss。
* **为什么公平：** 它有未来预测能力，能检验“预测后果是否自然产生 when-to-use”。
* **检验 claim：** necessity supervision 是否仍有独立贡献；world model 是否只改善视觉表示。
* **需要数据：** action-conditioned trajectories 或 action-free video + IDM。
* **最小实验：** action-only、video+action、video+action+necessity；控制 backbone 和数据。

## Baseline 7：LLM agent + skill library

* **做什么：** RoboTool/PLATO 风格，VLM/LLM 在 `direct_manipulate`、`fetch_tool`、`reposition_base`、`use_tool`、`abort` 技能间规划。
* **为什么公平：** 这是最直观的显式 when-to-use 方案，必须作为非端到端对照。
* **检验 claim：** 学习型策略是否在视觉状态变化、动作连续性、延迟和错误恢复上优于符号 agent。
* **需要数据：** 不需 policy training data，但需可靠技能、API、状态描述和执行器。
* **最小实验：** 使用相同 perception 和 low-level skills；比较 mode accuracy、task success、latency、错误传播。

# 推荐主实验矩阵

| Axis | Values |
|---|---|
| Task family | reach-extension / surface-contact / force-or-geometry transformation |
| Necessity class | N0 direct sufficient / N1 tool necessary / N2 tool unusable / N3 one of tools suitable |
| Data | success-only / unpaired negatives / paired counterfactual / paired + AIGC |
| Model | generic VLA / language-only / affordance-only / necessity-conditioned / WM+necessity |
| Embodiment | fixed arm simulation / quadruped-arm simulation / small real quadruped-arm subset |
| Tool-state intervention | presence / reachability / geometry / damage / target distance |

## 最小数据规模建议

先验证 claim，不追求 foundation-scale：

- 3 task families；
- 每类 40 个 base configurations；
- 每个 base configuration 4 个 matched variants；
- 480 个 scene states；
- 每状态 3–5 trajectories 或 1 个 decision label + planner-generated target；
- 真实测试每类 10 个 base configurations，至少 3 次重复。

统计单位必须是 **base scene**，而不是把同一 scene 的 counterfactual 当独立样本随机切分。训练/测试应按工具实例、布局或 base scene 分组，防止泄漏。

# Positive-negative / counterfactual sample design

## 标签定义

建议不要只用二分类：

| Label | Definition | Correct high-level mode |
|---|---|---|
| N0 Optional/unnecessary | direct action 可完成，工具增加成本或风险 | direct |
| N1 Necessary | direct action 不可行，至少一个工具可行 | acquire/use tool |
| N2 Unavailable | 任务需要工具，但当前工具不可用/不可达/损坏 | reposition/search/abort |
| N3 Selective | 多工具中只有部分满足功能/几何/鲁棒性 | select suitable tool |

也可以分解为三个可解释二值变量：

`direct_feasible`, `tool_available`, `tool_suitable`。

最终 mode 是这三个变量与成本的函数。该分解比单一“use_tool”标签更容易解释错误。

## 反事实生成原则

一次只改变一个因果变量：

- 目标距离跨过 direct reach boundary；
- 工具从 reachable 移到 occluded/unreachable；
- 工具 tip 形状从合适改为不合适；
- 工具完好改为断裂/缺失 functional part；
- 目标障碍高度改变，使 direct path 从可行变不可行；
- base standability 改变，使工具方案不可执行。

保持不变：

- task instruction；
- 目标类别与目标状态；
- 尽量一致的背景、光照、相机；
- 非干预对象的位置。

## 不应混入的“伪负样本”

- 随机换背景导致模型仅学视觉域；
- 把失败终帧直接当 no-tool negative，却没有相同初态；
- 把错误语言和正确图像混在一起；
- 生成视频中机器人形态已变化但仍沿用动作标签；
- 工具不存在和工具不必要混成一个类别。

# Reviewer-facing 实验

## 必须做的 ablation

1. 去掉 paired structure，只保留等量正负样本。
2. 去掉 necessity loss，只训练 action。
3. 去掉 tool state，只输入 RGB。
4. tool mask vs structured state。
5. oracle state vs estimated state vs noisy state。
6. simulation only vs simulation+AIGC vs real-only small set。
7. generic video-quality filter vs robot-specific filter。
8. fixed-base vs quadruped-arm，同一高层任务定义。

## 最能打消质疑的 intervention tests

- **State flip:** 只移动工具到不可达位置，模型应从 use-tool 改为 reposition/search，而非继续抓空。
- **Necessity flip:** 只把目标移入 direct workspace，模型应停止不必要取工具。
- **Suitability flip:** 工具外观相近但 functional tip 改变，模型应改变选择。
- **Cost flip:** 工具可用但非常远，direct 仍可行；模型应考虑获取成本。
- **Perception noise:** tool-state estimate 添加 pose/availability 噪声，报告 calibration 和 graceful degradation。

## 评价指标

| Level | Metrics |
|---|---|
| Decision | macro-F1, AUROC, ECE, N0↔N1 confusion |
| Counterfactual | correct mode-flip rate, pairwise ranking accuracy, invariance on non-causal edits |
| Action | waypoint/tool-tip error, trajectory DTW, action NLL |
| Mobile execution | task/partial success, collision, base travel, time, WBC tracking error |
| Tool use | unnecessary-use rate, missed-necessary-use rate, wrong-tool rate, acquisition failures |
| Data quality | verifier AUROC, pseudo-action error, accepted diversity, score-policy-gain correlation |
| Transfer | sim-to-real drop, unseen tool/layout/task-family success |

# 哪些 baseline 必须做

最低提交标准：

1. generic VLA；
2. language-only；
3. affordance-only；
4. unpaired simulation negatives；
5. paired simulation counterfactual；
6. world-model/action-only 或 future-prediction baseline；
7. LLM+skill baseline（可只做高层 decision，不必做同等规模训练）。

如果论文主打 AIGC，再必须加入：

- no filter；
- generic quality filter；
- same-count random subset；
- simulation-only；
- real-only。

# 贡献分层建议

一篇最小论文只保留一个主贡献层级：

1. **Mechanism contribution:** counterfactual necessity objective / structured tool-state conditioning；
2. **Evaluation contribution:** intervention protocol and metrics；
3. **Supporting data:** paired simulation/AIGC data；
4. **System support:** quadruped-arm + WBC。

不要把四层都写成同等级贡献。最稳妥的是 1 为主，2 为必要证据，3–4 为支撑。
