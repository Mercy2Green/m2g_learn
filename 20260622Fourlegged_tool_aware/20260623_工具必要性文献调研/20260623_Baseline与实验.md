# A. PPT-Level Baseline Taxonomy

这些 baseline 用于说明领域位置，不表示两周内全部实现。

| Baseline family | Representative route | What it tests | Stage |
|---|---|---|---|
| 1. No-tool-awareness VLA | OpenVLA / Octo | 通用 image-language representation 是否已能隐式判断 mode | 后续完整论文 |
| 2. Language-prompt-only | 显式 prompt “use/do not use tool” | 收益是否只是语言泄漏 | 组会 taxonomy；可做轻量版本 |
| 3. Affordance-recognition-only | RT-Affordance / AffordGrasp | necessity 是否超出 functional-region/how-to prediction | 后续完整论文 |
| 4. Simulation positive-negative | 独立随机正负、matched pairs | paired structure 是否有独立价值 | **第一阶段核心** |
| 5. AIGC positive-negative | unfiltered / generic-filtered / state-filtered | AIGC 是数据增广还是标签噪声来源 | Claim A 成立后 |

# B. First-Stage Experimental Baselines

两周内只保留四个模型，固定同一视觉 encoder、训练预算和数据切分。

## B0 — Image + Language Only

* 输入：当前 RGB、task instruction。
* 输出：四类 mode。
* 目的：测试模型是否仅凭场景像素和语言即可判断 necessity。
* 风险：可能学习距离、工具出现等 shortcut。

## B1 — Image + Language + Tool State

* 输入：B0 + `tool_present`, `tool_reachable`, `tool_usable`, `tool_suitable`。
* 输出：四类 mode。
* 目的：测试结构化状态是否有信息增益。
* 限制：不能区分信息增益与 oracle label leakage。

## B2 — Image + Language + Tool State + Paired Counterfactual Loss

* 输入与 B1 相同。
* 训练：classification loss + matched-pair consistency/ranking loss。
* 目的：测试 paired structure，而不只是更多特征或更多样本。
* 这是第一阶段 proposed method。

## B3 — Shuffled / Wrong Tool State

* 架构与 B1/B2 相同；
* 在训练或测试时打乱 tool state，或提供尺寸相同的随机 token；
* 目的：排除参数量增加、额外 token 和标签泄漏的解释。

# C. Minimal Two-Week Experiment

## C1. Research question

> 在固定任务定义和视觉分布下，matched counterfactual supervision 是否比同量独立样本提高 tool necessity 判别和 mode-change consistency？

## C2. Task scope

第一阶段只做两个可控因素，不同时研究复杂接触：

1. **Direct reachability intervention**
   - 目标在 direct workspace 内：`direct manipulate`
   - 目标移出 direct workspace，但工具可达且可延伸：`use tool`

2. **Tool reachability/availability intervention**
   - 工具可达且可用：`use tool`
   - 工具不可达但 base 可重定位：`reposition base`
   - 工具不可用且无可行替代：`abort / recovery`

暂不加入 hammering、sweeping、wrench 或复杂连续接触。

## C3. Data design

- 2 task factors；
- 30–50 个 base scenes；
- 每个 base scene 4 个 matched variants；
- 120–200 个 base-state groups，优先保证成对质量；
- 每个 variant 自动生成 oracle mode label；
- train/validation/test 按 base scene 分组，禁止同一 pair 跨 split；
- AIGC 暂不进入主实验。

## C4. Model

- 冻结轻量视觉 encoder 或使用预训练 VLM embedding；
- language encoder 固定；
- 2–3 层 MLP/Transformer mode head；
- 第一阶段不训练 OpenVLA action decoder；
- 第一阶段不输出连续 trajectory。

## C5. Go / No-Go Criteria

进入下一阶段需同时满足：

- B2 在 unseen base scenes 上显著优于 B1 和同量 unpaired training；
- counterfactual consistency 提升，而非只提升总体 accuracy；
- B3 明显退化，证明模型实际使用 tool state；
- 距离匹配 hard subset 上仍有收益。

# D. Metrics

## Primary metrics

| Metric | Why primary |
|---|---|
| Macro-F1 | 避免多数 mode 掩盖少数 mode |
| N0/N1 or direct/use-tool confusion | 直接测 necessity 核心混淆 |
| Counterfactual consistency | 测同一 base scene 状态变化时 mode 是否正确变化 |
| Mode-flip accuracy | 测指定 intervention 是否引发预期决策改变 |

## Secondary metrics

- Expected Calibration Error；
- unnecessary-tool-use rate；
- missed-necessary-tool rate；
- reposition-vs-abort confusion；
- hard-subset accuracy；
- non-causal visual edit invariance。

第一阶段不把 task success、trajectory DTW、WBC error 作为主指标，因为模型尚未执行完整 policy。

# E. Required Ablations

1. Paired loss 去除：B2 → B1。
2. Pair matching 打乱：保留样本量但破坏配对。
3. Tool state 打乱：B3。
4. Tool state 添加噪声：测试鲁棒性。
5. 同维随机 token：排除模型容量效应。
6. 距离匹配 hard pairs：排除“只学 reach threshold”。
7. Non-causal appearance edits：检查背景变化不应改变 mode。
8. Scene-group split vs random split：展示数据泄漏影响。

# F. Strongest Objections and Responses

| Objection | Required response |
|---|---|
| 模型只学目标距离 | 构造相同距离、不同工具可用性的 hard pairs；报告分层指标 |
| Structured state 是 oracle label leakage | wrong/shuffled/noisy state；estimated state 作为第二阶段 |
| Pair loss 只是增加训练样本 | 控制总样本数；unpaired same-count baseline |
| 四类 mode 由规则即可决定 | 承认规则是 oracle upper bound；研究问题是从视觉和不完美状态估计中泛化 |
| 仿真结果不能说明真实机器人 | 第一阶段只验证学习命题；真实小规模 state/mode test 是下一 gate |
| 与 affordance classification 没区别 | 加入“工具可用但不必要”和“工具合适但不可达”样本 |
| 不是完整机器人 policy | 明确这是 hypothesis validation，不包装为完整 policy paper |

# G. Later-Stage Experiments

仅在第一阶段成立后增加：

## Stage 2 — Estimated Tool State

- 从 RGB/depth 估计 tool presence、pose、reachability；
- 比较 oracle、estimated、noisy state；
- 验证 perception error 与 decision error 的边界。

## Stage 3 — VLA Integration

- 在 OpenVLA/Octo representation 上增加 mode head；
- 只在 mode 正确后调用 direct/tool/reposition skill 或 trajectory policy；
- 对比 image-only VLA 与 explicit-state-conditioned VLA。

## Stage 4 — AIGC Supporting Experiment

- 用 AIGC 扩外观和语义场景；
- 比较 unfiltered、generic-filtered、state-consistency-filtered；
- 不把 generated video 当作默认 action labels。

## Stage 5 — Quadruped Execution

- 固定 WBC；
- 测 mode 到执行接口；
- MoManipVLA/UMI on Legs/RoboDuet 只作为 execution/background reference。

# H. Contribution Boundary

| Layer | Role |
|---|---|
| Counterfactual necessity objective | Potential core contribution |
| Intervention evaluation protocol | Core evidence |
| Simulation scene generator | Supporting component |
| AIGC augmentation/filtering | Optional supporting variable or later Claim B |
| OpenVLA/Octo integration | Later validation |
| Quadruped WBC and base-arm control | System support |

第一阶段成功标准是证明一个可复现的 learning effect，不是完成四足臂工具任务 demo。
