# 2025–2026 Baseline 与实验设计

## 1. 现有 baseline 能否直接做四类 tool-aware mode decision？

| Existing baseline | Year | Code | Native input | Native output | Direct four-mode support? | Missing component |
|---|---:|---|---|---|---|---|
| Qwen3-VL | 2025 | [Yes](https://github.com/QwenLM/Qwen3-VL) | Image/video + prompt | Text/structured answer | **Near-runnable** | Tool-state dataset、prompt/schema、counterfactual evaluation |
| GR00T N1.7 | 2025–2026 | [Yes](https://github.com/NVIDIA/Isaac-GR00T) | Images + state + language | Continuous actions | No | Mode target/head、quadruped-arm modality adapter |
| π0.5/openpi | 2025 | [Yes](https://github.com/Physical-Intelligence/openpi) | Multi-view image + state + language | Action chunks | No | Mode token/head、LeRobot data conversion |
| Aff-Grasp | 2025 | [Yes](https://github.com/Reagan1311/Aff-Grasp) | RGB/ego video | Functional/grasp mask | No | Availability/reachability/necessity decision |
| RoboTwin 2.0 | 2025 | [Yes](https://github.com/robotwin-Platform/robotwin) | Task/scene/robot config | Labeled trajectories | No | Tool necessity tasks and quadruped embodiment |
| Video Policy | 2025 | [Yes](https://github.com/cvlab-columbia/videopolicy) | Robot video | Future video + action | No | Tool-state scenes and mode evaluation |
| UWM | 2025 | [Yes](https://github.com/WEIRDLabUW/unified-world-model) | Video/action conditions | Video and/or actions | No | Necessity labels and decision head |
| Human2LocoMan | 2025 | [Project release](https://human2bots.github.io/) | Vision + proprioception | Quadruped manipulation actions | No | Tool-state labels and mode output |
| MoManipVLA / SG-VLA | 2025/2026 | No public code found | Mobile observations + language | Base-arm action/trajectory | No | Runnable release and tool-aware state |

结论：

> 2025–2026 年未找到可直接输出本项目四类 mode 的公开 baseline。第一阶段应以可运行的 Qwen3-VL reasoning baseline 为主；GR00T/openpi 是后续 action-policy baseline；RoboTwin 2.0/Isaac Sim 提供数据生成基础。任何自定义 classifier 只能标记为 `custom sanity check, not existing baseline`。

## 2. A. Runnable / Near-Runnable Baseline Tests

### A1. Qwen3-VL Four-Mode Reasoning

* **Baseline:** Qwen3-VL, 2025
* **Code:** https://github.com/QwenLM/Qwen3-VL
* **Weights:** Qwen3-VL-2B/4B/8B Instruct or Thinking
* **Input:** 单帧或多视角图像 + task instruction + 可选结构化 tool state
* **Output:** 强制 JSON：`mode`, `direct_feasible`, `tool_available`, `tool_reachable`, `tool_suitable`, `reason`
* **Required modification:** Prompt template、JSON parser、matched pair sampler；可选 LoRA/SFT
* **Runnable this week:** Yes
* **Why fair:** 2025 开源成熟 VLM；不需自行发明视觉 backbone
* **Tests:** 通用视觉推理是否已经支持四类 mode；matched examples是否提高 intervention consistency
* **Cannot test:** Robot action executability

实验：

1. zero-shot neutral prompt；
2. few-shot independent examples；
3. few-shot matched counterfactual examples；
4. wrong/shuffled tool-state prompt；
5. optional Qwen3-VL-4B/8B LoRA if prompting shows signal。

### A2. Aff-Grasp Affordance-Only Baseline

* **Baseline:** Precise Affordances / Aff-Grasp, ICCV 2025
* **Code:** https://github.com/Reagan1311/Aff-Grasp
* **Data/weights:** Repository-linked Hugging Face releases
* **Input:** RGB / egocentric data
* **Output:** Functional/graspable masks
* **Required modification:** 将 mask/functional label提供给同一 Qwen3-VL prompt；不提供 reachability/availability
* **Runnable this week:** Partial；取决于环境安装
* **Why fair:** 直接检验“affordance information 是否足够”
* **Tests:** tool function/how-to baseline versus full tool-state input
* **Cannot test:** Base reachability and tool necessity by itself

### A3. GR00T N1.7 Open-Loop Baseline

* **Baseline:** Isaac GR00T current N1.7; lineage from GR00T N1 2025
* **Code:** https://github.com/NVIDIA/Isaac-GR00T
* **Input:** Image/state/language/embodiment config
* **Output:** Continuous relative actions
* **Required modification:** 首先只跑官方 demo和自定义 observation schema；四类 mode 作为 auxiliary target 属后续
* **Runnable this week:** Partial；需要 16GB inference / 40GB recommended tuning
* **Why fair:** 2025–2026 可运行 robot foundation model
* **Tests:** 当前 policy是否可被适配；不作为两周核心结果
* **Cannot test:** Necessity without new labels/head

### A4. RoboTwin 2.0 Data Generation Baseline

* **Baseline:** RoboTwin 2.0, 2025
* **Code:** https://github.com/robotwin-Platform/robotwin
* **Input:** Task config, embodiment, scene randomization
* **Output:** RGB/depth/state/action trajectories
* **Required modification:** 新增 tool/no-tool scene variables、oracle feasibility和四类 mode label
* **Runnable this week:** Yes for platform smoke test；quadruped migration is not immediate
* **Why fair:** 现有 2025 simulation data generator，不是自造数据 pipeline
* **Tests:** 官方 generator能否承载 project-specific counterfactual extension
* **Cannot test:** 四足臂 tool-awareness without extension

## 3. B. Data / Motivation Experiment

### B1. Simulation Counterfactual Dataset Extension

* **Existing source:** RoboTwin 2.0 2025；若平台不匹配，使用项目已有 Isaac Sim，但明确标记为 project-specific extension。
* **Required variables:** direct feasibility、tool presence、tool reachability、tool usability、tool suitability、base reposition feasibility。
* **Output label:** direct / use tool / reposition / abort-recovery。
* **Pairs:** 每个 base scene 只改变一个变量。
* **Important wording:**  
  `2025–2026 年未找到可直接复用的公开 tool-necessity pair generator；该数据是基于现有 simulation framework 的项目扩展，不是 existing baseline。`

### B2. Qwen3-VL Counterfactual Evaluation

| Condition | Existing baseline connection | Claim tested |
|---|---|---|
| Zero-shot Qwen3-VL | Qwen3-VL 2025 | Mature VLM without project data |
| Independent few-shot | Qwen3-VL prompt baseline | More examples versus pair structure |
| Matched-pair few-shot/SFT | Qwen3-VL + project simulation data | Counterfactual supervision effect |
| Aff-Grasp mask only | Aff-Grasp 2025 | Affordance versus necessity |
| Wrong/shuffled state | Qwen3-VL control | Whether state is actually used |

Primary metrics：

- four-mode macro-F1；
- direct/use-tool confusion；
- pairwise mode-flip consistency；
- tool-state contradiction rate；
- unnecessary-tool-use rate；
- distance-matched hard subset accuracy。

### B3. AIGC Supporting Test — Only After B2

* **Existing sources:** DreamGen、Video Policy、UWM、Cosmos-Predict2。
* **Input:** Simulation-rendered matched scenes。
* **Output:** Appearance/semantic variants；第一阶段不采用 generated action。
* **Comparisons:** original sim / unfiltered generated / generic-quality filtered / tool-state-label filtered。
* **Claim:** 过滤后的 AIGC 是否改善 Qwen3-VL 或后续 GR00T/openpi 的视觉泛化。
* **Required verification:** tool presence、geometry、reachability cue、task result、pair label preservation。

## 4. Later Policy / Mobile Experiments

### C1. π0.5/openpi

- 将 simulation/real data 转为 LeRobot；
- 添加 mode token/auxiliary prediction或 hierarchical mode conditioning；
- 仅在 Qwen3-VL counterfactual结果成立后进行；
- 公平检验 paired data 对 action policy 的增益。

### C2. GR00T N1.7

- 增加 quadruped-arm modality/embodiment config；
- 比较 no explicit state 与 tool-state-conditioned data；
- 需要足够 GPU 和真实 action data；
- 不作为周末后两周任务。

### C3. Mobile / Quadruped Execution

- **Human2LocoMan:** 复用数据、模型和 embodiment interface；
- **MoManipVLA:** 只作 waypoint/base-arm conceptual baseline，因未找到公开代码；
- **SG-VLA:** 迁移 spatial auxiliary supervision；
- **MLM/SLIM:** whole-body/sim-to-real background；
- 固定执行层，避免把 controller 改进当 tool-aware 改进。

## 5. Required Ablations

1. Qwen3-VL zero-shot vs independent few-shot vs matched-pair few-shot。
2. Tool-state fields removed。
3. Tool state shuffled/wrong。
4. Aff-Grasp affordance only vs full state。
5. Same-count independent versus paired data。
6. Distance-matched and appearance-matched hard pairs。
7. Original sim versus AIGC unfiltered/filtered（后续）。
8. Qwen3-VL versus GR00T/openpi only after policy integration。

## 6. Go / No-Go

**Go：**

- Qwen3-VL matched condition improves pairwise mode consistency over independent examples；
- shuffled state causes a clear drop；
- gains remain on distance-matched scenes；
- simulation labels can be audited automatically。

**No-go：**

- only random-split accuracy improves；
- state shuffling has no effect；
- labels are determined by a trivial visible distance threshold；
- generated visuals frequently change tool-state labels。

## 7. C. Not to Do Now

- 不训练新的通用 VLM。
- 不从零造一个 classifier并称为 strong baseline；如使用，只能写 `custom sanity check, not existing baseline`。
- 不把任何 pre-2025 模型放入主实验。
- 不直接使用 AIGC pseudo-actions。
- 不训练 UWM/Video Policy/DreamGen 大模型作为第一阶段。
- 不做完整 WBC、trajectory optimizer 或 agent+skill system。
- 不把 Gemini Robotics 写成可复现 baseline。
