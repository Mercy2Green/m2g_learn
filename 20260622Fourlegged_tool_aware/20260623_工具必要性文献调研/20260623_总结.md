# 总结（Cleaned）

## One-Sentence Project Proposition

> 研究四足臂移动操作中，模型能否根据 direct feasibility 与工具的可用性、可达性和合适性，在 direct manipulate、use tool、reposition base、abort/recovery 之间做出正确模式选择。

## Current Best Minimal Research Claim

> 在移动操作任务中，成对反事实的 tool/no-tool、usable/unusable、reachable/unreachable 样本，相比同量非配对数据，可以提升 tool necessity 判别和工具状态变化下的 mode-decision consistency。

这是第一阶段 hypothesis，不是完整论文结论。两周内先验证离散 mode，不主张连续轨迹、完整 VLA 或闭环工具使用。

## Why This Is Not Just Tool Recognition

- 工具识别回答“这是什么”；
- affordance 回答“它能怎么用”；
- 本项目回答“不用工具是否可行、当前工具是否可执行、应切换到哪个 mode”；
- 核心证据是 matched intervention 后正确 mode flip，而不是 detection accuracy。

## Why This Is Not Just AIGC Data Generation

- 仿真负责可控物理变量、counterfactual pairs 和 mode labels；
- AIGC 只作为语义/外观扩展候选；
- AIGC 数据必须检查语义、tool state、任务结果和标签一致性；
- generated video 不默认等于 action supervision；
- 第一阶段主实验不依赖 AIGC。

## Tier 1 Papers

1. Creative Robot Tool Use by Counterfactual Reasoning  
   用途：counterfactual tool-property reasoning。
2. Robustness-Aware Tool Selection and Manipulation Planning  
   用途：工具状态和配置影响 robust choice，但不等于 necessity。
3. Physics-Conditioned Grasping for Stable Tool Use  
   用途：证明 tool physics 会改变 downstream grasp/action。
4. RT-Affordance  
   用途：定义 affordance/how-to 与 necessity/whether 的边界。
5. AHA  
   用途：程序化 failure/negative synthesis。
6. Dream2Fix  
   用途：paired generated failures 和 structured verification。

Tier 1 用于构造 motivation 和反对意见；OpenVLA、Octo、DreamGen、UWM 等属于 Tier 2 baseline；MoManipVLA、UMI on Legs、RoboDuet 属于 Tier 3 系统支撑。

## Corrected Research Gap

> 现有机器人工具使用工作主要研究工具选择、affordance grounding、task-oriented grasp、tool-use trajectory 和 contact robustness。较少工作系统构造同一任务下 direct manipulation 与 tool-use 的成对反事实对照，并评测模型是否根据工具可用性、可达性和合适性改变高层 mode decision。

不使用“没人做过”“完全空白”“四足工具使用是全新问题”等表述。

## First-Stage Experiment

### Inputs

- RGB；
- task instruction；
- optional structured tool state。

### Outputs

- direct manipulate；
- use tool；
- reposition base；
- abort / recovery。

### Baselines

1. image + language；
2. image + language + tool state；
3. image + language + tool state + paired counterfactual loss；
4. shuffled/wrong tool state。

### Primary Metrics

- macro-F1；
- direct/use-tool confusion；
- counterfactual consistency；
- mode-flip accuracy。

### Required Controls

- same-count unpaired samples；
- distance-matched hard pairs；
- shuffled/noisy state；
- base-scene grouped split。
