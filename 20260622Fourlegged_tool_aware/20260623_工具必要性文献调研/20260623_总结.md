# Executive Summary

检索截止：2026-06-23。

## 一句话结论

当前最可发表、最容易证伪的命题不是“做一个会用工具的四足 VLA”，而是：

> **用成对的 tool/no-tool、usable/unusable、reachable/unreachable 反事实样本，训练一个 tool-necessity-conditioned policy；证明工具必要性和工具状态会因果性地改变 mode、末端轨迹和底盘—机械臂协同，而不只是提高工具识别准确率。**

## 对 10 个核心问题的短答

1. **现有工作主要是 how，而不是 when。**  
   主流研究解决选哪个工具、抓哪里、如何规划、如何保持接触鲁棒性。把“直接用手/夹爪”作为平等候选并学习是否需要工具的工作很少。

2. **明确研究 tool necessity 的强证据不足。**  
   RoboTool 等构造“无工具不可行”的任务，Learning at the Ends 讨论 hand-to-tool affordance，近期 counterfactual tool-use 工作研究工具物理属性；但都没有形成标准的 tool/no-tool necessity 学习与评测范式。

3. **正负/失败/反事实数据已被用于机器人学习，但极少用于 tool necessity。**  
   AHA/FailGen、Guardian、Dream2Fix 用扰动或生成模型合成失败；Creative Robot Tool Use by Counterfactual Reasoning 扰动工具属性做因果发现。缺口是同一任务、同一目标下的“无工具成功/失败、工具可用/损坏/不可达”成对监督。

4. **AIGC 当前主要有三种用途。**
   - 改外观：ROSIE、GenAug、EMMA。
   - 生成视觉计划或行为先验：Gen2Act、RoboDreamer、GR-2。
   - 生成带伪动作的训练轨迹或失败：DreamGen、Video Policy、UWM、Dream2Fix。

5. **AIGC 的主要问题不是单一的“视频不够真”。**  
   更关键的是 action grounding、接触物理、时序/多视角一致性、机器人形态漂移、标签可信度和负样本覆盖。视觉上真实不等于可执行。

6. **world model 能提供未来结构，但不能自动等价于“意图”或“何时行动”。**  
   只有当模型受任务、动作、状态约束，并通过 action decoder、inverse dynamics 或决策损失连接到控制，且在反事实决策指标上验证时，才能声称帮助了决策。

7. **VLA 可以隐式利用工具状态，但现有证据通常不证明它真的用了。**  
   大多数工作没有 tool-state intervention：替换工具位置、可达性、损坏状态后，检查 mode/trajectory 是否按预期改变。因此“看到了工具”和“以工具状态决定动作”没有被区分。

8. **四足臂新增 failure mode。**
   - 工具、臂和机身的联合可达性；
   - base reposition 与 arm motion 的时序耦合；
   - 支撑稳定性和反作用力；
   - 工具改变碰撞包络；
   - 机身振动/足端滑移导致工具端误差；
   - locomotion/manipulation/tool-use mode 切换；
   - 视角变化和自遮挡；
   - 工具获取成本可能高于直接操作。

9. **必须做的 baseline。**  
   OpenVLA/Octo 类无工具意识 VLA、语言显式提示、affordance-only、AIGC 正负、仿真正负、world-model future prediction、LLM+skill library。

10. **最适合最小论文的命题。**  
    首选“counterfactual tool necessity learning”；次选“AIGC 样本的机器人可执行性筛选”；风险最高的是“大一统四足 tool-aware VLA 系统”，容易成为工程 demo。

## 三个候选 claim 的优先级

| Priority | Claim | 可证明性 | 主要风险 | 推荐 venue |
|---|---|---:|---|---|
| 1 | 反事实 tool/no-tool 数据能提升必要性判断和 mode/trajectory 选择 | 高 | 数据对是否真正隔离变量 | ICRA / IROS / RA-L |
| 2 | 物理与动作一致性筛选比通用视频质量筛选更能预测 AIGC 数据价值 | 中高 | 需足够生成数据和真实下游验证 | CoRL / ICRA |
| 3 | 显式 tool state conditioning 会因果性改变 VLA 输出并提高四足臂成功率 | 中 | 容易被质疑为多加输入或系统集成 | CoRL / TRO / RA-L |

## 最小实验定义

任务不宜超过三族：

1. **Reach-extension**：目标在直接 arm workspace 外，但在“工具端 + 合理 base pose”内。
2. **Surface-contact**：擦、扫、刮；工具姿态和接触面状态决定可行性。
3. **Force/geometry transformation**：钩、撬、推、舀；直接夹爪或错误工具会系统失败。

每个初始场景生成四类配对：

- `N0`: 不用工具可成功，工具是多余成本；
- `N1`: 不用工具失败，合适工具可成功；
- `N2`: 工具在场但不可用/不可达/损坏；
- `N3`: 多工具在场，仅部分工具合适。

最少输出：

- necessity 四分类或二分类；
- mode：direct / acquire-tool / reposition / abort-or-replan；
- 末端或 tool-tip waypoint；
- 可选 base goal。

## 最关键指标

- Necessity AUROC / macro-F1，重点报告 N0 与 N1 的混淆；
- Counterfactual consistency：只改变工具状态时决策改变是否正确；
- Mode accuracy 与 unnecessary-tool-use rate；
- Task success、partial success、collision、tool acquisition cost；
- Tool-state intervention 下 trajectory displacement / mode flip；
- sim-to-real degradation；
- AIGC 样本质量分数与真实 policy gain 的相关性。

## 哪些只是支撑，不应写成主贡献

- IsaacSim 场景搭建；
- Cosmos/Sora-like 视频生成调用；
- WBC 或 local controller；
- 通用数据清洗 pipeline；
- 单纯收集一个小型数据集；
- 单纯把 tool mask、tool token 拼到 VLA 输入；
- 组装 VLM planner + skill library。

除非这些组件带来一个可泛化、可消融、可复现的新机制，否则只能作为实现与实验支撑。

## 我这周最应该读的 10 篇

1. [Creative Robot Tool Use by Counterfactual Reasoning](https://arxiv.org/abs/2605.05411)  
   最接近“通过反事实扰动物理属性建立工具因果关系”，用于界定你的反事实设计与其差异。
2. [Robustness-Aware Tool Selection and Manipulation Planning with Learned Energy-Informed Guidance](https://arxiv.org/abs/2506.03362)  
   最强的工具选择+接触鲁棒性对照，说明“选择工具”仍不等于“判断是否需要工具”。
3. [Physics-Conditioned Grasping for Stable Tool Use / iTuP](https://arxiv.org/abs/2505.01399)  
   证明工具状态和任务 wrench 会改变抓取与执行，是 tool-state-to-action claim 的直接依据。
4. [AHA: A Vision-Language-Model for Detecting and Reasoning Over Failures](https://arxiv.org/abs/2410.00371)  
   学习如何从成功轨迹程序化生成失败，并把失败监督接入决策。
5. [Learning Actionable Manipulation Recovery via Counterfactual Failure Synthesis](https://arxiv.org/abs/2603.13528)  
   最新的生成式反事实失败数据与严格筛选范式，直接支撑 Candidate 2。
6. [DreamGen: Unlocking Generalization in Robot Learning through Neural Trajectories](https://arxiv.org/abs/2505.12705)  
   了解视频世界模型生成数据、伪动作恢复和下游有效性相关评测。
7. [Unified World Models](https://arxiv.org/abs/2504.02792)  
   最适合理解“视觉未来 + 动作”如何统一，以及为何纯视频预测不足。
8. [RT-Affordance](https://arxiv.org/abs/2411.02704)  
   强 affordance intermediate-representation baseline；可检验必要性是否超出 affordance。
9. [MoManipVLA](https://arxiv.org/abs/2503.13446)  
   最贴近 base-arm feasibility 的 mobile manipulation baseline。
10. [UMI on Legs](https://arxiv.org/abs/2407.10353)  
    最贴近四足臂“高层末端轨迹 + WBC”接口，帮助把主贡献和底层控制支撑拆开。

排序原则：先读能定义 novelty 的工具/反事实工作，再读数据生成和 world model，最后读与你硬件接口最接近的 mobile manipulation。

## 两周建议

第一周：

- 固定 necessity label ontology 和三类任务；
- 在仿真中做 30–50 个基础场景，每个场景生成 N0–N3 配对；
- 先训练轻量分类/排序头，不先训练大 VLA；
- 用 intervention test 验证模型是否真的依赖 tool state。

第二周：

- 加入一个开源 VLA backbone 的 LoRA/adapter；
- 对比 random negatives、unpaired negatives、counterfactual pairs；
- 做 10–20 个真实场景的小规模 sanity check；
- 只有在仿真结果证明必要性信号有效后，再投入 AIGC 视频生成和 WBC 集成。
